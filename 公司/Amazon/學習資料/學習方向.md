# Amazon - 學習方向

> Amazon SDE 準備分三大支柱。與多數外商不同的是：**Behavioral（Leadership Principles）權重極高**，幾乎每一輪都會問，務必與 Coding 同等投入。

## 🎯 三大準備支柱

### 1. Coding（DSA） — 基礎、佔輪數最多
- Amazon coding 題以**資料結構與演算法**為主，難度多為 LeetCode Easy~Medium，少數 Hard。
- 高頻主題：Arrays/Hashing、Two Pointers、Sliding Window、Trees/BFS-DFS、Graphs、Heap、DP、Intervals。
- 目標：能在 35~40 分鐘內釐清題意、講解思路、寫出可執行且處理 edge case 的程式，並分析複雜度。

### 2. System Design — L5（SDE II）以上必考
- 中高級職位必考；L4 可能出現簡化版或物件導向設計（LLD）。
- 重點：可擴展性、可用性、一致性取捨、實際估算（容量/QPS/儲存）。

### 3. Behavioral（Leadership Principles） — Amazon 的決勝點 ⭐
- **16 條 Leadership Principles**，每一輪面試官都會挑幾條來問。
- 需要一個**結構化的 STAR 故事庫**，且每個故事能對應多條 LP。
- **Bar Raiser** 一關特別深挖 LP、追問細節（會連續問「然後呢？」「你具體做了什麼？」「數據是多少？」）。
- 詳見 [面試準備/行為面試.md](../面試準備/行為面試.md)。

## 📚 DSA 主題路線（依序）
```
Arrays & Hashing → Two Pointers → Sliding Window → Stack →
Binary Search → Linked List → Trees → Tries → Heap / Priority Queue →
Backtracking → Graphs → Advanced Graphs → 1-D DP → 2-D DP →
Greedy → Intervals → Math & Geometry → Bit Manipulation
```
- 建議跟著 **NeetCode 150** 的主題順序逐塊攻克，每塊先看模式再刷題。
- 具體題單見 [學習資料/練習題目.md](./練習題目.md)。

## 🏗️ System Design 重點
- **核心概念**：Scalability（垂直/水平擴展）、Load Balancing、Caching（cache 策略、失效）、DB sharding/replication、SQL vs NoSQL、CAP 定理、一致性模型、Message Queue（解耦/非同步）、CDN、Rate Limiting、Idempotency。
- **AWS 加分**：熟悉 DynamoDB、S3、SQS/SNS、Lambda、ELB、CloudFront、Aurora 等對應概念（Amazon 內部偏好用自家服務舉例）。
- **推薦資源**：
  - The System Design Primer（GitHub）
  - 《Designing Data-Intensive Applications, DDIA》（Martin Kleppmann）
  - Grokking the System Design Interview
  - System Design / LLD（物件導向設計）練習

## 🗣️ Behavioral（LP）重點
- 整理 **8~12 個 STAR 故事**，標註每個故事可覆蓋哪些 LP。
- 每個故事 2~3 分鐘，用 **「I」（我）而非「We」（我們）** 描述個人貢獻，並量化結果。
- 預先準備被深挖的細節：衝突如何處理、數據從哪來、若重來會怎麼做、失敗教訓。
- 詳細逐條對應表見 [面試準備/行為面試.md](../面試準備/行為面試.md)。

## 🗓️ 準備時程（範例：8~10 週）
| 週次 | Coding | System Design | Behavioral (LP) |
|------|--------|---------------|-----------------|
| W1-2 | Arrays/Hashing、Two Pointers、Sliding Window、Stack | 讀 Primer 概念 | 盤點過往專案，草擬 6 個故事 |
| W3-4 | Binary Search、Linked List、Trees、Tries | Caching、DB、Queue | 用 STAR 寫成稿，對應 LP |
| W5-6 | Heap、Backtracking、Graphs | 練 2~3 題經典設計 | 擴充到 10~12 故事、錄音自測 |
| W7-8 | DP、Greedy、Intervals | 完整 mock design | LP mock（含追問演練） |
| W9-10 | 複習錯題、Amazon Tagged 高頻題 | mock design × 2 | 完整 loop mock（coding+LP） |
- **每日**：2~3 題 coding + 朗讀/打磨 1 個 LP 故事。
- **每週**：至少 1 次 mock interview（coding 或 LP）。
