# Amazon - 練習題目

> **以具體題單為主**。Amazon coding 以 LeetCode 為主，難度集中在 Easy~Medium、少數 Hard。
> 題號/題名以 LeetCode 為準；難度標示 E=Easy、M=Medium、H=Hard。

## 📋 精選題單
- **Blind 75**：經典 75 題，覆蓋所有核心模式，第一輪刷題首選。
- **NeetCode 150**：Blind 75 的擴充版（含更多 DP/Graph），題單與影片講解：https://neetcode.io/practice
- **Amazon Tagged 高頻題**：LeetCode 上以 `Amazon` 標籤篩選、依「最近 6 個月 / 頻率」排序（需 LeetCode Premium，或參考社群整理）。
- **Grind 75**：Blind 75 的可調進度版本（grind75.com），可依時間自動排程。

## 🏗️ System Design 練習題（L5+ 必練）
- **設計 TinyURL / URL Shortener**（hashing、redirect、讀多寫少快取）
- **設計 Rate Limiter**（token bucket / sliding window、分散式計數）
- **設計 News Feed / Timeline**（fan-out on write vs read、快取）
- **設計 Chat / Messaging System**（WebSocket、訊息順序、已讀回執）
- **設計 Web Crawler**（去重、排程、禮貌爬取）
- **設計 Notification Service**（SQS/SNS、retry、idempotency）
- **設計 Top K / Trending**（heap、count-min sketch、串流處理）
- **設計 Distributed Key-Value Store / DynamoDB**（一致性雜湊、replication、quorum）
- **設計 Parking Lot / Elevator（LLD/OOD）**（物件導向設計，L4 常見）
- **設計 Amazon 電商相關**：商品搜尋、購物車、訂單系統、庫存（貼合 Amazon 業務情境）

## 🗓️ 刷題策略
- **先廣後深**：先把 NeetCode 150 / Blind 75 各主題模式刷一遍，再針對弱項深掘。
- **計時**：每題限時（Medium 約 25~30 分），模擬面試壓力。
- **複習錯題**：用間隔重複（如 1 天 / 3 天 / 7 天）回顧卡關題。
- **邊寫邊講**：練習 think-aloud，講解思路與複雜度（面試評分重點）。
- **Mock interview**：用 Pramp / interviewing.io 找人對練，貼近真實 loop。
- **收尾衝刺**：面試前 1~2 週主刷 Amazon Tagged 高頻題。
