# Amazon - 履歷準備

## 📄 履歷原則（外商）
- **英文、一頁**（資深者最多兩頁，但一頁優先）。
- **XYZ 公式**：Accomplished **[X]** as measured by **[Y]**, by doing **[Z]**。
  - 例：*Reduced API p99 latency by 40% (Y) by redesigning the caching layer with Redis (Z), improving checkout success (X)*。
- **量化成果**：效能提升 %、延遲下降、QPS/流量規模、用戶數、成本節省（$）、可用性（9 的數量）。
- **ATS 友善**：標準字體、無表格/圖片/雙欄、關鍵字對齊 JD（程式語言、框架、雲端服務）。

## 🅰️ 對齊 Amazon Leadership Principles
Amazon 連履歷與招募系統都偏好能體現 LP 的描述。撰寫 bullet 時刻意嵌入 LP 訊號：
- **Customer Obsession**：強調成果如何讓客戶/使用者受益。
- **Ownership**：端到端負責（設計→上線→維運/on-call）。
- **Deliver Results**：在期限/限制下交付可量化結果。
- **Dive Deep**：找出根因、用數據佐證（不要只寫「優化效能」，要寫測量與根因）。
- **Invent and Simplify**：簡化系統、移除複雜度、自動化。
- **Bias for Action / Frugality**：用小成本/快速行動達成（精簡資源、節省成本）。

## 🔗 內推 (Referral)
- **優先找內推**，命中率遠高於海投；可透過 LinkedIn 找 Amazon 員工（同校/同事/社群）。
- **LinkedIn 經營**：標題寫清楚技術棧與目標、定期更新專案、與招募人員（recruiter）建立連結。
- Amazon 官方招募網站：[amazon.jobs](https://www.amazon.jobs/)；可設定 job alert。
