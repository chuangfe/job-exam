# Amazon - SDE（Software Development Engineer）總覽

> 範本類型：**外商科技**（LeetCode + 系統設計 + 行為面試）。
> Amazon 特色：**Leadership Principles（LP）行為面試比重極高**，每一輪面試官幾乎都會問 LP。

## 🏢 公司簡介
- **公司名稱**：Amazon（亞馬遜，含 AWS）
- **產業 / 規模**：電商 + 雲端運算（AWS）+ 數位內容，全球員工數十萬人規模的大型科技公司。
- **在台據點**：AWS 在台有辦公室與資料中心（台北），亦有部分研發/解決方案架構職缺；多數 SDE 核心職缺位於美國、加拿大、歐洲、印度等地。
- **文化關鍵字**：**Leadership Principles（16 條領導力準則）**、Customer Obsession、Ownership、Day 1、Bar Raiser、Frugality、Dive Deep。

## 💼 目標職位
- **職位**：SDE（Software Development Engineer，軟體開發工程師）
- **常見職級**：
  - SDE I（L4，新鮮人/初階）
  - SDE II（L5，資深，2~5 年經驗，最常見招募職級）
  - SDE III / Senior SDE（L6，資深，需有系統設計與技術領導能力）
- **工作內容**：設計、開發、測試、部署與維運大規模分散式系統與服務；參與 on-call、code review、技術設計（design doc）；以客戶為中心交付高品質可擴展的服務。

## 🔄 面試流程（關卡）
1. **履歷篩選 / 內推（Referral）** — 內推命中率明顯較高，建議優先。
2. **線上評測（OA, Online Assessment）**
   - Coding（通常 1~2 題演算法，自動評測 + 隱藏測資）
   - Work Simulation / Work Style Assessment（情境模擬與工作風格測驗，對應 Leadership Principles）
   - 部分職級含 Logical/Debugging 題組
3. **電話面（Phone Screen）** — 1 輪，coding（線上協作編輯器）+ 穿插 LP 行為問題。
4. **On-site / Virtual Loop（核心關卡）** — 通常 **4~5 輪**，每輪約 45~60 分鐘：
   - Coding × 2~3（DSA）
   - System Design × 1（L5 以上必考；L4 視情況）
   - Behavioral / LP（穿插於各輪，幾乎**每一輪都會問 LP**）
   - **Bar Raiser** × 1：由跨團隊資深面試官擔任，重點把關 LP 與長期潛力，擁有否決權。
- **時程**：從投遞/內推到 offer 通常數週至 1~2 個月；loop 後約 1~2 週出結果（含 debrief 會議）。

## 💰 薪資結構
> Amazon 薪資由三部分組成，**RSU 前兩年較低、靠 sign-on bonus 補足**（典型「前低後高」結構）。
- **Base Salary（底薪）**：有上限（部分地區/職級有 base cap）。
- **RSU（限制性股票）**：4 年 vesting，但 Amazon 慣用 **5%/15%/40%/40%** 的非均勻釋出（前兩年較少）。
- **Sign-on Bonus（簽約金）**：通常分兩年發放（Year 1、Year 2），用以補足前兩年 RSU 較低的缺口。
- **參考來源**：實際數字依職級、地區、議價而定，請查 [levels.fyi](https://www.levels.fyi/companies/amazon/salaries) 取得各職級（L4/L5/L6）的市場區間，**不要憑記憶捏造精確數字**。
