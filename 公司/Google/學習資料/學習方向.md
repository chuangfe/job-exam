# Google SWE - 學習方向

## 🎯 三大準備支柱

### 1. Coding（DSA）— 佔比最高
Google 面試以**資料結構與演算法**為核心，幾乎所有輪次都圍繞 coding。
- 重點能力：寫出**乾淨、可執行**的程式；**邊想邊講**（think aloud）；主動做**複雜度分析**（time / space）；處理 edge cases；寫完能自行測試。
- 語言：選一個最熟的（Python / Java / C++ / Go），求快且正確。

### 2. System Design — 中高級職位（L5+）必考
- 初階（L3/L4）通常以 coding 為主，但理解基礎概念有加分。
- 資深需展現：估算、API 設計、高階架構、瓶頸與 trade-off 溝通。

### 3. Behavioral（Googleyness & Leadership）— 文化契合
- 對應 Googleyness：協作、面對模糊、使用者導向、謙遜好奇、主動。
- 用 STAR 準備故事庫，至少一輪會深入問行為與過往專案。

## 📚 DSA 主題路線（依序學習）
```
Arrays & Hashing → Two Pointers → Sliding Window → Stack →
Binary Search → Linked List → Trees → Tries → Heap / Priority Queue →
Backtracking → Graphs → Advanced Graphs → 1-D DP → 2-D DP →
Greedy → Intervals → Math & Geometry → Bit Manipulation
```
> 此即 NeetCode 150 的主題順序，先廣度建立每個主題的「招式」，再回頭加深難題。

### 各主題該掌握的核心技巧
- **Arrays & Hashing**：用 HashMap/Set 換取 O(1) 查找、前綴和。
- **Two Pointers / Sliding Window**：有序陣列收斂、可變/固定視窗。
- **Stack**：單調堆疊（monotonic stack）、括號匹配。
- **Binary Search**：在「答案空間」二分、找邊界。
- **Trees**：DFS（前/中/後序）、BFS（層序）、遞迴思維。
- **Heap**：Top-K、合併 K 個串列、中位數。
- **Backtracking**：排列/組合/子集、剪枝。
- **Graphs**：BFS/DFS、拓樸排序、Union-Find、Dijkstra。
- **DP**：辨識狀態與轉移、由暴力遞迴 → 記憶化 → 自底向上。

## 🏗️ System Design 重點（資深）
- 核心觀念：Scalability、Load Balancing、Caching、DB 分庫分表（sharding）、Replication、CAP 定理、Message Queue、一致性 vs 可用性取捨。
- 推薦資源：
  - **System Design Primer**（GitHub）
  - **DDIA**《Designing Data-Intensive Applications》
  - **Grokking the System Design Interview**

## 🗣️ Behavioral 重點
- 建立 STAR 故事庫（見 [行為面試](../面試準備/行為面試.md)），每則對應一項 Googleyness。
- 練習把同一個專案故事拆出不同面向（領導 / 衝突 / 失敗 / 主動）。

## 🗓️ 準備時程（建議 8-12 週）
| 階段 | 週數 | 內容 |
|------|------|------|
| 基礎 | 第 1-4 週 | NeetCode 150 前半主題（Arrays~Trees），每日 2-3 題 |
| 進階 | 第 5-8 週 | Graphs / DP / Backtracking，每日 2-3 題 + 開始 mock |
| 衝刺 | 第 9-12 週 | 複習錯題、計時模擬、Google tagged 高頻題、System Design + 行為故事 |
- 每週至少 1 次 **mock interview**（Pramp / interviewing.io / 同伴）。
