# Google SWE - 行為面試

## 🗣️ STAR 架構
- **S**ituation（情境）/ **T**ask（任務）/ **A**ction（你的行動）/ **R**esult（結果，量化）。
- 每個故事控制在 **2-3 分鐘**，重點放在「**我**做了什麼」（用 I 而非 we），結尾帶量化結果與反思。
- 準備英文版，講順為止。

## 🎯 Googleyness & Leadership 對應
> Google 不像 Amazon 有逐條 Leadership Principles，而是綜合評估 **Googleyness**（文化契合）與 **Leadership**（影響力）。以下面向各準備 1-2 則 STAR 故事。

| Googleyness 面向 | 我的故事（STAR 摘要） |
|------------------|----------------------|
| 協作 / 團隊合作（Collaboration） | [跨團隊合作交付某功能…] |
| 面對模糊（Comfort with ambiguity） | [需求不明確時如何釐清並推進…] |
| 使用者導向（User-focus） | [以使用者體驗為出發點做的決策…] |
| 謙遜與好奇心（Humility & curiosity） | [主動學新技術 / 接受回饋改進…] |
| 主動 / Bias to action（Ownership） | [沒人負責時主動扛下並解決…] |
| 領導力 / 影響他人（Leadership） | [非主管身分帶動他人、推動改變…] |
| 處理衝突（Conflict） | [與同事意見不合如何達成共識…] |
| 從失敗學習（Failure） | [專案失誤、如何補救與學到什麼…] |

## 📚 故事庫（建議準備 8-12 個）
列出可重複利用的專案故事，標註可對應的面向：
1. [最具技術挑戰的專案] — 影響、領導、面對模糊
2. [跨團隊協作專案] — 協作、衝突
3. [一次失敗 / 重大 bug] — 失敗、謙遜
4. [主動發起的改善] — 主動、領導
5. [以使用者為中心的決策] — 使用者導向
6. [時間壓力下交付] — 主動、優先排序
7. [學習新技術解決問題] — 好奇心
8. [說服他人 / 影響決策] — 領導、溝通
> 一個好專案可拆成多個面向，避免每題都換故事。

## ❓ 常見問題
- Tell me about yourself.
- 你做過最有挑戰 / 最自豪的專案？
- 與同事（或主管）意見不合時你怎麼處理？
- 講一次失敗的經驗，你學到什麼？
- 如何在需求模糊的情況下推進工作？
- 一次你主動發起改變的經驗。
- 你如何接受並回應負面回饋？
- 為什麼想加入 Google？

## 💡 加分技巧
- 結尾加上**反思 / 學到什麼**，展現成長心態（謙遜 + 好奇）。
- 用數據量化結果（效能、時間、人數、營收）。
- 故事真實、細節具體，面試官會追問細節。
