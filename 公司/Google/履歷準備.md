# Google SWE - 履歷準備

## 📄 履歷原則（外商）
- **英文、一頁**（新鮮人/中階一頁；資深可放寬到兩頁但仍力求精簡）。
- **XYZ 公式**：Accomplished **[X]** as measured by **[Y]**, by doing **[Z]**。
  - 例：Reduced API latency by 40% by introducing a Redis caching layer serving 2M requests/day.
- **量化成果**：效能提升 %、用戶/流量規模、節省成本、QPS、資料量級——有數字才有說服力。
- **ATS 友善**：關鍵字對齊 JD（語言、框架、系統）、簡潔單欄格式、避免表格/圖片造成解析失敗。
- **動詞開頭**：Built / Designed / Led / Optimized / Shipped…，避免 "Responsible for"。

## 🎯 對齊 Google JD
- 凸顯**大規模系統 / 分散式 / 效能**經驗。
- 展現**協作與影響力**（跨團隊、code review、mentor）對應 Googleyness。
- 技術棧寫真正熟的，面試會被追問。

## 🔗 內推 (Referral)
- **優先找內推**，命中履歷篩選的機率遠高於海投。
- 透過 **LinkedIn** 經營：找在 Google 的校友/前同事，禮貌詢問是否願意內推。
- 內推時附上：精簡自我介紹、目標職位連結、一頁英文履歷。
- 投遞官方職缺：**careers.google.com**。

- [ ] 對齊目標職位 JD 關鍵字
- [ ] 動詞開頭、無 "Responsible for"
- [ ] 單欄、ATS 可解析格式（PDF）
