# 中華郵政（資訊類）- 面試／面談指南

> ⚠️ 中華郵政資訊類**通常以筆試成績為主要錄取依據**，是否有口試／面談、其比重多寡，**視該梯次官方簡章而定**。
> 本檔提供「若該梯次有面試」時的準備方向；若該梯次純筆試，請把時間全投入專業科目。

## 🎯 面試概述（若有）

- **可能形式**：個人面談、口試或書面審查＋簡短問答（依梯次）。
- **比重**：通常低於筆試，甚至無；**以簡章為準**。
- **重點**：確認人格特質、表達能力、對職務與公司的理解、專業基礎。

## 📋 可能流程

```
報到（提早 20-30 分鐘）
    ↓
等候叫號 → 進入面談室
    ↓
自我介紹（約 1 分鐘）
    ↓
委員提問（專業＋動機＋特質）
    ↓
應試者反問（1-2 題）
    ↓
離開
```

## 💼 準備重點

### 自我介紹（1 分鐘）
架構：開場（姓名／學歷）→ 資訊相關修課與專題／實務經驗 → 技能與證照 → 報考中華郵政的動機（穩定、希望投入郵政核心系統開發與維運）。

範例骨架：
```
各位委員好，我是 ___，畢業於 ___ 大學資訊工程／資管系。
在學期間修習資料結構、資料庫、計算機網路、資訊安全等課程，
專題以 ___ 技術開發 ___ 系統，負責 ___。
我熟悉 ___（語言／資料庫／網路），具備 ___ 證照。
我希望加入中華郵政資訊單位，以穩健的態度參與郵務／儲匯等
核心系統的開發與維運，並重視系統穩定與資訊安全。
```

### 專業問題（鎖定兩大科目）
- 資料結構：Stack 與 Queue 差異？BST 何時用？排序複雜度比較？
- 資料庫：正規化目的？INNER 與 LEFT JOIN 差異？ACID 是什麼？
- 網路：TCP 與 UDP 差異？OSI 七層？子網路怎麼算？
- 資安：對稱與非對稱加密差異？SQL Injection 如何防護？CIA 三要素？
- 結合郵政情境：金融通路為何特別重視資安與系統穩定？

### 行為問題（STAR）
以「情境→任務→行動→結果」回答團隊合作、解決問題、學習新技術的經驗。

### 公司與動機
- 為何選中華郵政（穩定、國營、郵政＋金融通路規模）。
- 對郵政數位轉型（行動支付、APP、資料中心）的了解。
- 能接受部分核心系統的傳統技術（主機／COBOL）並穩紮穩打。
- 職涯規劃：朝資訊開發或維運／資安深耕。

## 👔 服裝儀態
- 正式、整潔、保守（深色西裝／套裝）。
- 準時（提早到）、禮貌、眼神接觸、聲音清晰、手機靜音。

## ⚠️ 常見失分原因
- 履歷／自我介紹寫的東西答不出來。
- 基本專業觀念不清。
- 對中華郵政與郵政數位轉型毫無了解。
- 態度問題：遲到、不禮貌、過度緊張或自負。
