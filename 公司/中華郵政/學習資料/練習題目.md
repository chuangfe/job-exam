## 📝 高頻考點題型範例（含簡短解析）

### 資料結構 — 運算式轉換
```
【範例】將中序式 A + B * C - D 轉為後序式（Postfix）。
【解析】
 乘除優先於加減：B*C 先做 → BC*
 由左而右：A + (BC*) → ABC*+，再 - D → ABC*+D-
 答案：A B C * + D -
（技巧：用堆疊處理運算子，遇較低優先運算子先彈出。）
```

### 資料結構 — 排序複雜度
```
【範例】下列何者最差情況時間複雜度為 O(n²)？
(A) Merge Sort (B) Heap Sort (C) Quick Sort (D) 以上皆否
【解析】(C)。Quick Sort 平均 O(n log n)，但若每次 pivot 取到最大／最小，
 退化為 O(n²)。Merge、Heap 皆穩定 O(n log n)。
（延伸：Quick 是 in-place 但不穩定；Merge 穩定但需 O(n) 額外空間。）
```

### 資料結構 — 樹遍歷還原
```
【範例】已知前序 = A B D E C F、中序 = D B E A C F，求後序。
【解析】
 前序首位 A 為根；中序中 A 左側 (D B E) 為左子樹、右側 (C F) 為右子樹。
 遞迴拆解後，後序 = D E B F C A
（重點：前序定根、中序分左右子樹，是必練手法。）
```

### 資料庫 — 正規化
```
【範例】關聯 R(學號, 課號, 課名, 成績)，函數相依：
 學號,課號 → 成績；課號 → 課名。此關聯違反第幾正規化？
【解析】違反 2NF。主鍵為 (學號,課號)，但「課號 → 課名」是
 非主屬性對「部分主鍵」的相依（部分相依）。
 分解：R1(學號,課號,成績)、R2(課號,課名) 即達 2NF/3NF。
```

### 資料庫 — SQL JOIN
```
【範例】員工表 emp(id, name, dept_id)、部門表 dept(dept_id, dept_name)，
 要列出「所有員工」及其部門名稱，沒有部門者部門名顯示 NULL，該用哪種 JOIN？
【解析】LEFT JOIN（以 emp 為左表全保留）：
 SELECT e.name, d.dept_name
 FROM emp e LEFT JOIN dept d ON e.dept_id = d.dept_id;
（INNER JOIN 會漏掉沒部門的員工，故不適用。）
```

### 網路 — 子網路計算（必考）
```
【範例】192.168.10.0/26 可分成幾個子網路、每網段可用主機數？
【解析】/26 主機位元 = 32-26 = 6，每網段位址 2^6 = 64，
 可用主機 = 64 - 2 = 62（扣網路位址與廣播位址）。
 由 /24 切到 /26 多借 2 位元 → 2^2 = 4 個子網路。
 網段：.0~.63、.64~.127、.128~.191、.192~.255。
```

### 網路 — TCP 三方交握
```
【範例】TCP 建立連線的三方交握順序為何？
(A) SYN → SYN-ACK → ACK
(B) SYN → ACK → SYN-ACK
【解析】(A)。Client 送 SYN → Server 回 SYN-ACK → Client 回 ACK，連線建立。
（對照：連線結束為四次揮手 FIN/ACK。）
```

### 資安 — 攻擊辨識
```
【範例】攻擊者在登入欄位輸入 ' OR '1'='1 試圖繞過驗證，屬於何種攻擊？
 應如何防護？
【解析】SQL Injection（資料隱碼攻擊）。
 防護：使用參數化查詢（Prepared Statement）、輸入驗證、
 最小權限資料庫帳號、避免動態字串組 SQL。
```

### 資安 — 加密觀念
```
【範例】下列敘述何者正確？
(A) 對稱式加密用同一把金鑰加解密，速度快但金鑰分送是難題
(B) RSA 屬對稱式加密
(C) 雜湊函數可逆，能還原原文
【解析】(A) 正確。RSA 為非對稱式；雜湊為單向不可逆（用於完整性／密碼儲存）。
```

### 國文 — 公文（共同科目高 CP 值）
```
【範例】公文「函」的正文採三段式，依序為哪三段？
【解析】主旨 → 說明 → 辦法。
 「主旨」扼要敘明目的與期望（含期望語如「請鑒核」「請查照」）；
 「說明」交代來龍去脈；「辦法」提出具體要求。
（背熟格式幾乎穩拿分，務必反覆默寫。）
```
