## 🎯 資訊系統開發及維運 - 準備心得

### 【PTT心得】資訊系統開發及維運準備方向討論

**來源**：PTT Gov_owned板，2023年10月

#### 考試科目
- 計算機概論
- 程式設計概論

#### 準備時間
- 距離考試不到2個月
- 多數考生認為時間足夠

#### 重點整理

**1. 計算機概論**
```
難度：★★★☆☆
範圍：廣但不深

考試內容：
✓ 資料結構基礎（前中後序轉換）
✓ 作業系統（排程、佇列）
✓ 基本概念題（很多是常識）
✓ 英文題目（單字、文法較深）

準備建議：
- 不需補習，自讀即可
- 範圍雖廣但深度不高
- 基本觀念理解為主
- 有經驗者「裸考應該都沒問題」
```

**2. 程式設計概論**
```
難度：★★☆☆☆
語言：不限定

考試內容：
✓ 巢狀迴圈程度
✓ 基本演算法
✓ 程式閱讀題

準備建議：
- 語言沒有特別限制
- 難度不會太高
- 熟悉基本語法即可
```

**3. 英文**
```
難度：★★★☆☆（2024年後降低）

考試內容：
✓ 單字較深
✓ 文法題
✓ 閱讀測驗

準備建議：
- CP值不高，不用花太多時間
- 2024年改由「松盟」出題後難度降低
```

**4. 出題單位變更**
```
⚠️ 重要變更（2024年）：
- 改由「松盟」出題
- 英文難度顯著降低
- 專業科目題型大致不變
- 考古題參考價值存疑（但仍有重複題）
```

#### 考生建議
1. **不需補習**
   - 有基礎的考生自讀即可
   - 計概不會很難

2. **重點在面試**
   - 「筆試相對不難」
   - 「面試標準較謎」
   - 通過筆試才是關鍵

3. **練習其他考試題目**
   - 雖然中華電信沒有釋出考古題
   - 可練習其他國營考試題目
   - 國營聯招資訊類題目很類似

### 【Dcard心得】資訊系統規劃及開發 - 具工作經驗上岸

**來源**：Dcard國營版，2024年

#### 考試流程時間軸
```
D+21   第一次面試通知
D+28   筆試 + 第一次面試
D+78   第二次面試通知
D+83   專業面試 + HR面試
D+99   Offer通知
```
總耗時約3個多月

#### 筆試內容（30分鐘）

**考試形式**
- 選擇題 + 英文寫作
- 時間：30分鐘
- 地點：公司會議室

**題目內容**
1. **Java與OOP概念**（40分）
   - 多為選擇題
   - 考基本觀念
   - 難度：中等

2. **英文寫作題**（60分）
   ```
   主題包括：
   ✓ Dependency Injection（依賴注入）
   ✓ Secure Coding Practices（安全編碼實務）
   ✓ Git指令應用
   ```

#### 面試內容

**第一次面試（D+28）**
- 5分鐘簡報
- 30分鐘Q&A

```
簡報主題（自選）：
- 專題/畢業論文
- 工作經驗
- 專案經驗

Q&A主題：
✓ 專案管理手法
✓ DevOps概念與實作
✓ 資訊安全知識
✓ 技術細節深入討論
```

**第二次面試（D+83）**

*專業面試*
```
問題類型：
✓ 工作經驗細節
✓ 使用的技術棧
✓ 專案遇到的挑戰
✓ 如何解決問題
✓ 對新技術的看法
```

*HR面試*
```
問題類型：
✓ 為何想加入中華電信
✓ 對職位的理解
✓ 職涯規劃
✓ 期望薪資（已有標準起薪）
✓ 何時可報到
```

#### 準備建議
1. **專業知識**
   - Java OOP基礎必須熟悉
   - 了解DevOps概念
   - 資安基礎知識
   - 專案管理經驗

2. **英文能力**
   - 技術英文寫作能力
   - 能用英文解釋技術概念
   - 常見技術術語

3. **面試準備**
   - 準備簡報檔案
   - 整理專案經驗
   - 準備STAR故事
   - 了解中華電信近況

## 💡 綜合建議（基於多位考生心得）

### 筆試準備

#### 新鮮人（無工作經驗）
```
準備重點：
1. 紮實的計算機概論基礎
2. 熟練至少一種程式語言
3. 大量練習程式題
4. 了解新技術趨勢（5G、IoT、雲端）

優勢：
✓ 剛學過相關課程，記憶新鮮
✓ 理論基礎較紮實

劣勢：
✗ 缺乏實務經驗
✗ 面試較難發揮

建議：
- 強調學校專題經驗
- 展現學習熱忱
- 補充實習或side project經驗
```

#### 有工作經驗者
```
準備重點：
1. 複習計算機概論（可能生疏）
2. 整理工作專案經驗
3. 準備技術面試
4. 了解中華電信業務

優勢：
✓ 實務經驗豐富
✓ 面試較容易發揮
✓ 對技術理解較深

劣勢：
✗ 理論可能生疏
✗ 準備時間較少（需上班）

建議：
- 利用零碎時間複習
- 整理STAR故事
- 強調可立即上手
```

### 面試準備

#### 關鍵要素（重要性排序）
```
1. 專業知識  ★★★★★
   - 基本觀念要清楚
   - 能說明專案技術細節
   - 了解新技術趨勢

2. 表達能力  ★★★★☆
   - 邏輯清晰
   - 說話有條理
   - 不緊張、有自信

3. 工作態度  ★★★★☆
   - 積極主動
   - 願意學習
   - 團隊合作

4. 企業了解  ★★★☆☆
   - 知道中華電信在做什麼
   - 了解5G等業務發展
   - 說得出報考動機

5. 英文能力  ★★☆☆☆
   - 基本技術英文即可
   - 不是決定性因素
```

#### 常見失敗原因
```
❌ 筆試階段：
1. 計算機概論基礎不紮實
2. 程式設計能力不足
3. 完全沒準備（裸考失敗）

❌ 面試階段：
1. 過度緊張，表現失常
2. 專業問題答不出來
3. 對中華電信毫無了解
4. 履歷寫的東西說不出來
5. 態度問題（不禮貌、太傲慢）
```

## 🎯 具體準備步驟（考生推薦）

### 第一階段：基礎建立（2-3個月）

**Week 1-4：資料結構與演算法**
```
□ 陣列、鏈結串列、堆疊、佇列
□ 樹（二元樹、BST）
□ 排序演算法（Quick, Merge, Heap Sort）
□ 搜尋演算法（Binary Search, DFS, BFS）
□ 時間複雜度分析
```

**Week 5-8：作業系統**
```
□ 程序管理與排程
□ 記憶體管理（Paging, Virtual Memory）
□ 死結（Deadlock）
□ 檔案系統
□ I/O管理
```

**Week 9-12：資料庫與網路**
```
□ ER Model與正規化
□ SQL語法（DDL, DML, DCL）
□ Transaction與ACID
□ OSI七層與TCP/IP
□ 常見網路協定
```

### 第二階段：深入學習（1個月）

**Week 13-14：計算機組織**
```
□ CPU結構與運作
□ 記憶體階層
□ Cache原理
□ I/O系統
```

**Week 15-16：資訊安全與新技術**
```
□ 密碼學基礎
□ 常見攻擊與防護
□ 5G技術與應用
□ 雲端運算（IaaS/PaaS/SaaS）
□ IoT基礎
```

### 第三階段：實戰演練（1個月）

**Week 17-18：考古題練習**
```
□ 國營聯招資訊類歷屆試題
□ 高普考資訊處理歷屆試題
□ 限時模擬考試
□ 錯題整理與複習
```

**Week 19-20：程式實作**
```
□ LeetCode Easy 50題
□ LeetCode Medium 20題
□ APCS歷屆試題
□ 手寫程式練習
```

### 第四階段：衝刺準備（考前1週）

**考前7天**
```
□ 複習所有筆記
□ 快速翻閱重點章節
□ 背誦重要公式
□ 不碰新內容
```

**考前3天**
```
□ 輕鬆複習
□ 維持正常作息
□ 準備考試用品
□ 確認考場位置
```

**考前1天**
```
□ 早點睡覺
□ 不要讀書
□ 放鬆心情
□ 準備隔天物品
```
