# 國營聯招（資訊類師級）- 第二試（口試）準備指南

> ⚠️ 第二試形式（純口試／含專業測驗）與佔比**逐年、依承辦單位與分發機構而異**，以**當年度簡章**為準。本指南以「個別口試」為主。

## 🎯 面試概述
- **方式**：多為個別口試（部分年度可能加入專業實作或筆試型測驗）。
- **佔比**：第二試常見約佔總成績 20%（第一試筆試約 80%）；**確切比例依簡章**。
- **重要性**：筆試是主力，但口試足以影響正取／備取與分發順位，不可輕忽。
- **評分重點**：專業知識、表達邏輯、工作態度與穩定性、對國營事業的認識。

## 📋 面試流程（典型）
```
收到第二試通知（第一試錄取後）
    ↓
提早 20-30 分鐘報到、抽序號
    ↓
等候區候場
    ↓
進入口試室 → 自我介紹（約 1 分鐘）
    ↓
口試委員提問（專業 + 經歷 + 態度，約 5-10 分鐘）
    ↓
（部分年度）反問時間
    ↓
離開，等待總成績與分發公告
```

## 💼 準備重點

### 1. 自我介紹（1 分鐘）
**架構**：開場（姓名、學校科系）→ 專業背景（修過的核心課程、專題）→ 經歷／證照 → 報考動機（為何選國營、可接受全台分發）。

**範例（新鮮人）**：
```
各位委員好，我是王小明，畢業於○○大學資訊管理學系。
在學期間修習資料庫系統、系統分析與設計、資訊安全等核心課程，
專題以 Spring Boot 與 MySQL 開發校園選課系統，由我負責資料庫設計與後端 API。
我已通過 TOEIC 750 分，並自學 SQL 調校與基礎資安防護。
報考國營聯招是希望在穩定的環境長期發展，也願意配合分發到台電／中油等機構服務。謝謝。
```

### 2. 專業問題（依資訊類核心科目）
**資料庫**
- Q：INNER JOIN 與 LEFT JOIN 差在哪？
- Q：正規化的目的？3NF 解決什麼問題？
- Q：交易的 ACID 是什麼？

**資訊管理**
- Q：請說明系統開發生命週期（SDLC）的階段。
- Q：瀑布式與敏捷開發的差異？各適合什麼情境？
- Q：ERP、CRM、SCM 分別是什麼？

**資通安全**
- Q：對稱式與非對稱式加密的差異？
- Q：SQL Injection 如何發生？如何防護？
- Q：你對 ISO 27001 或個資法有什麼了解？

**計算機概論 / 經驗**
- Q：行程（Process）與執行緒（Thread）的差別？
- Q：你做過最有挑戰的專案是什麼？負責什麼？怎麼解決困難？

> 回答原則：**先給結論，再簡短舉例／說明原理**；不會的題目誠實說明已知部分並表達學習意願，不要硬掰。

### 3. 行為問題（STAR）
- Q：團隊合作中意見不合如何處理？
- Q：在時間壓力下完成任務的經驗？
- Q：你如何學習新技術？

**STAR 範例（解決困難）**：
```
S：專題系統壓力測試時，逾百人同時上線即當機。
T：我負責後端，須在一週內解決並上線 Demo。
A：用效能分析工具定位瓶頸，發現查詢未建索引、快取設定錯誤，
   重寫查詢、建立索引並調整快取策略。
R：系統可支援 500 人同時上線，回應時間由 3 秒降至 0.5 秒內。
```

### 4. 國營事業 / 機構相關
- Q：為什麼想進國營事業？為什麼選資訊類？
- Q：你了解台電／中油的業務嗎？資訊部門可能做什麼？
- Q：可以接受分發到外縣市嗎？
- **準備**：報考前先看分發機構（台電／中油／台水／台糖）官網與近期新聞，能講出該機構資訊業務（如台電智慧電網、營業系統）會大加分。

## 👔 服裝儀態
- 深色正式服裝（西裝／套裝），整潔保守。
- 進門問好、委員示意再坐；眼神接觸、聲音清晰、坐姿端正、手機關機。
- 結束起立道謝再離開。

## ⚠️ 常見失分原因
- ❌ 遲到、服裝不整、態度敷衍。
- ❌ 履歷／自介寫的東西答不出來。
- ❌ 基本專業概念（SQL、SDLC、加密）講不清楚。
- ❌ 對國營事業與分發機構毫無了解。
- ❌ 表達不出可接受分發的意願，讓委員質疑穩定性。
- ❌ 回答過短（只答「是／不是」）或過長離題。
