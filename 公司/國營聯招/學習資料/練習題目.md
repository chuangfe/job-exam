## 📝 高頻考點題型範例

> 每個重點科目給代表題 + 解析，幫助你掌握出題深度。題目為示範用途，**正式準備請以官方歷屆與題庫為主**。

### 資料庫應用 — SQL（必考）
```
【範例】有資料表 Employee(emp_id, name, dept_id, salary)，
        請列出「每個部門平均薪資高於 50000」的部門編號與平均薪資。

【解答】
SELECT dept_id, AVG(salary) AS avg_salary
FROM Employee
GROUP BY dept_id
HAVING AVG(salary) > 50000;

【解析】篩選「群組後的條件」要用 HAVING，不能用 WHERE
        （WHERE 在分組前過濾列，HAVING 在分組後過濾群組）。
        這是 SQL 最高頻的觀念區分點。
```

### 資料庫應用 — 正規化（必考計算題）
```
【範例】資料表 R(學號, 姓名, 課號, 課名, 成績)，
        其中 學號→姓名、課號→課名、(學號,課號)→成績。
        此表最高符合第幾正規化？如何拆解到 3NF？

【解析】
- 主鍵為 (學號, 課號)。
- 「學號→姓名」「課號→課名」屬於「部分相依」（只依賴主鍵的一部分）
  → 違反 2NF。
- 拆解到 3NF：
   學生(學號, 姓名)
   課程(課號, 課名)
   選課(學號, 課號, 成績)
- 拆解後消除了插入/刪除/更新異常。
```

### 資訊管理 — 專案管理（要徑法 CPM）
```
【範例】專案有 A(3天)→C，B(2天)→C，C(4天)→D，B→D，D(2天) 為終點。
        （A、B 為起點）求要徑（critical path）與最短完工時間。

【解析】
- 路徑 A→C→D = 3+4+2 = 9 天
- 路徑 B→C→D = 2+4+2 = 8 天
- 路徑 B→D     = 2+2   = 4 天
- 最長路徑即要徑 = A→C→D = 9 天 → 最短完工時間 9 天。
  要徑上的作業沒有寬裕時間（slack=0），延誤會直接延後整個專案。
```

### 資訊管理 — 系統分析（DFD / 系統類型）
```
【範例】下列何種資訊系統主要服務「高階主管做非結構化決策」？
(A) TPS  (B) MIS  (C) DSS  (D) EIS

【解答】(D) EIS（主管資訊系統）

【解析】TPS 處理日常交易、MIS 產生例行管理報表、
        DSS 支援半結構化決策、EIS/ESS 服務高階主管的策略性決策。
        記住「管理層級 vs 系統類型」對應關係是必考點。
```

### 資通安全 — 密碼學
```
【範例】RSA 屬於下列何種加密？其相較於 AES 的主要優缺點為何？

【解析】
- RSA 為「非對稱式加密」，使用公鑰加密、私鑰解密（或反之做數位簽章）。
- 優點：解決金鑰交換問題、可做數位簽章與不可否認性。
- 缺點：運算速度慢、不適合加密大量資料。
- 實務常用「RSA 交換對稱金鑰 + AES 加密資料」的混合方式。
```

### 資通安全 — 常見攻擊
```
【範例】程式直接把使用者輸入拼接進 SQL 字串，
        導致攻擊者可竄改查詢邏輯，這是何種攻擊？如何防護？

【解答】SQL Injection（SQL 隱碼攻擊）。

【解析】防護：使用參數化查詢（Prepared Statement）、
        輸入驗證與最小權限資料庫帳號、ORM、WAF。
        「原理 + 防護」要能成對作答。
```

### 計算機概論 — 作業系統（排程，可手算）
```
【範例】Round Robin 排程，時間量子 = 4ms，
        P1(12ms)、P2(8ms)、P3(4ms) 同時於 t=0 到達（順序 P1,P2,P3），
        求 P2 的完成時間。

【解析】
時間軸：P1(0-4) P2(4-8) P3(8-12) P1(12-16) P2(16-20) P1(20-24)
P2 在 t=20 完成 → 完成時間 20ms。
```

### 計算機概論 — 資料結構（運算式轉換）
```
【範例】將中序式 A + B * C - D 轉為後序式（Postfix）。

【解答】A B C * + D -

【解析】依運算子優先序：* 高於 + 與 -，
        先處理 B*C → BC*，再 A + (BC*) → ABC*+，最後 - D → ABC*+D-。
        樹遍歷、運算式轉換為高頻題。
```

### 計算機概論 — 網路
```
【範例】TCP 三向交握（Three-way Handshake）的封包順序為何？

【解答】SYN → SYN-ACK → ACK
        1. Client→Server：SYN
        2. Server→Client：SYN-ACK
        3. Client→Server：ACK
        建立連線後才開始傳資料；UDP 則無此程序。
```
