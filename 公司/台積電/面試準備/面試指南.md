# 台積電 IT / 資訊系統工程師 - 面試準備指南

> 流程與題型依職缺/部門略有差異，**以官方通知為準**。台積電重視扎實技術、誠信、邏輯與抗壓。

## 🔄 面試關卡（常見）
1. **線上測驗 / 性向測驗**：邏輯推理、數理、英文閱讀；部分職缺含程式/技術筆試。
2. **技術面試（一次或多次）**：用人主管 + 資深工程師，技術觀念 + 實作 + 專案深掘。
3. **主管面試**：團隊適配、工作態度、抗壓性、解決問題思路。
4. **HR 面試**：動機、人格特質、生涯規劃、薪資談判。

## 🧠 性向 / 線上測驗準備
- **題型**：數列推理、邏輯圖形、文字理解、基礎數理、英文閱讀。
- **準備**：找一般性向測驗題本限時練習；測驗多有時間壓力，練「快又準」。
- 部分技術職含**程式/SQL 筆試**，先把練習題目中的 LeetCode Easy/Medium 與 SQL 題練熟。

## 💻 技術面試
- **常見題型**：
  - **語言觀念**：所選主力語言（Python/Java/C#）的記憶體管理、OOP、例外處理、集合型別差異、（Java）GC、（Python）GIL 等。
  - **資料結構與演算法**：手寫或口述解題（陣列/雜湊/樹/圖/DP），說明時間/空間複雜度。
  - **資料庫 / SQL**：JOIN、索引、交易 ACID、正規化、查詢優化（製造業資料量大常被問）。
  - **作業系統 / 網路**：行程 vs 執行緒、死結、TCP vs UDP、HTTP 流程。
  - **Debug / 實務**：給一段有問題的程式找 bug；如何排查線上問題；如何做 code review。
- **白板 / 線上 coding 注意事項**：
  - 先**釐清題意與邊界條件**再動手，講出思路（口語化思考）。
  - 先提暴力解再優化，說明複雜度。
  - 邊寫邊解釋，主動測試範例與 edge case。
- **專案深掘**（重點）：
  - 能講清楚做過的專案：問題背景、你的角色、技術選型理由、遇到的困難與如何解、量化成果。
  - 預期追問：「為何選這技術？」「效能瓶頸在哪、怎麼解？」「如果資料量放大 100 倍怎麼辦？」
  - 用練習題目中的 3 個專案，事先寫好可以講 3–5 分鐘的版本。

## 🤝 行為面試（STAR）
> 用 **Situation 情境 → Task 任務 → Action 行動 → Result 成果（量化）** 講故事。準備至少 4–5 個故事庫。

- **團隊合作**：與不同背景同事協作完成系統的經驗。
- **解決問題 / 抗壓**：時間緊迫或重大故障時如何處理（台積電很看抗壓）。
- **衝突處理**：與同事/跨部門意見不合如何溝通達成共識。
- **失敗 / 學習**：一次失敗經驗與你學到什麼、如何改進。
- **主動性**：自發改善流程、自學新技術的例子。

## 🏢 公司相關（事先想好）
- **為何選台積電**：對半導體龍頭、智慧製造/大規模 IT 系統的興趣，連結個人技能與職涯目標。
- **對產業/公司的了解**：晶圓代工角色、台積電核心價值（誠信、承諾、創新、客戶信任）、IT 在製造中的重要性。
- **誠信**：台積電極度重視 Integrity；回答務必誠實，技能不誇大（會被技術面拆穿）。

## ❓ 反問問題（準備 2–3 個）
- 這個職位/團隊目前最重要的技術挑戰是什麼？
- 團隊使用的技術棧與開發流程（Git/CI-CD/Code Review 文化）如何？
- 新人前 3–6 個月的學習與成長路徑、會接觸哪些系統？
- 部門如何與產線/其他單位協作？（展現對製造業 IT 的理解）
