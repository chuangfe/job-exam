# 求職追蹤庫 (job-exam)

個人求職與考試準備的知識庫。集中管理多家公司的職位資訊、學習方向、練習題目與面試準備。

## 📂 目錄結構

```
job-exam/
├── 說明.md              # 本檔案：專案介紹與使用方式
├── 求職追蹤.md          # 所有公司一覽表（類型 / 職位 / 時程 / 準備進度）
├── CLAUDE.md            # 給 AI 助手的維護規範
├── 練習題/              # 個人化練習題（依使用者背景設計，5 個軌道）
├── 範本/                # 三種公司類型的資料夾範本
│   ├── 考試制範本/      # 國營 / 公職招考
│   ├── 民間企業範本/    # 履歷 + 技術面試
│   └── 外商範本/        # LeetCode + 系統設計 + 行為面試
└── 公司/                # 各公司資料（每家自給自足）
    ├── 中華電信/        # 考試制
    ├── 國營聯招/        # 考試制
    ├── 中華郵政/        # 考試制
    ├── 台積電/         # 民間
    ├── Google/         # 外商
    └── Amazon/         # 外商
```

## 🧭 公司類型與資料夾內容

| 類型 | 適用 | 核心資料夾 |
|------|------|-----------|
| **考試制** | 中華電信、國營聯招、中華郵政、公股銀行 | 考試資訊 / 學習資料 / 考古題 / 面試準備 |
| **民間企業** | 台積電、一般軟體公司 | 總覽(JD) / 學習資料 / 履歷準備 / 面試準備 |
| **外商科技** | Google、Amazon | 總覽(流程) / 學習資料 / 履歷準備 / 面試準備(技術+行為) |

每家公司資料夾都包含 **學習方向** 與 **練習題目**，列出具體該學什麼、該練哪些題。

## 🚀 使用方式

1. 先看 [求職追蹤.md](./求職追蹤.md) 掌握全局與各公司準備進度。
2. 進入目標公司資料夾，從 `總覽.md` 開始。
3. 依 `學習資料/學習方向.md` 排讀書計畫，用 `學習資料/練習題目.md` 練題。
4. 動手練習用 [練習題/](./練習題/說明.md)（個人化、含完整解答的 5 個軌道）。
5. 進到面試階段再讀 `面試準備/`。

## ➕ 新增一家公司

判斷類型 → 從 `範本/` 複製對應範本到 `公司/<公司名>/` → 填入內容 → 在 `求職追蹤.md` 新增一列。詳見 [CLAUDE.md](./CLAUDE.md)。

---
**注意**：所有招考時程、薪資、JD 等資訊以各公司官方公告為準，本庫僅供準備參考。
