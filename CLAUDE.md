# CLAUDE.md

給 AI 助手（Claude）維護本專案的指引。

## 專案性質

這是**個人求職追蹤庫**（純 Markdown，無程式碼），集中管理多家公司的職位資訊、學習方向、練習題目與面試準備。**全程使用繁體中文（台灣用語）。**

## 目錄結構與慣例

```
說明.md          # README（專案介紹）
求職追蹤.md      # 所有公司一覽表（新增公司務必同步更新）
CLAUDE.md        # 本檔
範本/            # 三種類型範本：考試制 / 民間企業 / 外商
公司/<公司名>/   # 每家公司自給自足
docs/            # 設計文件（spec）
```

- 公司資料夾**自給自足**，不在根目錄放單一公司的專屬資料。
- 檔名用繁體中文、與既有檔案一致（總覽.md、學習方向.md、練習題目.md…）。
- 每份文件結尾標注 `**最後更新**：YYYY-MM-DD`。

## 三種公司類型

| 類型 | 適用 | 對應範本 |
|------|------|---------|
| 考試制 | 國營/公職招考（中華電信、國營聯招、郵政、公股銀行） | `範本/考試制範本/` |
| 民間企業 | 履歷+技術面試（台積電、軟體公司） | `範本/民間企業範本/` |
| 外商科技 | LeetCode+系統設計+行為面試（Google、Amazon） | `範本/外商範本/` |

## 新增一家公司的 SOP

1. **判斷類型**（考試制 / 民間 / 外商）。
2. 從對應的 `範本/` 複製整個資料夾到 `公司/<公司名>/`。
3. 將範本內 `[...]` 佔位符換成實際內容。
4. **重點**：`學習資料/學習方向.md` 與 `練習題目.md` 要有**實質內容**——
   - 學習方向：列出該學什麼、順序、深度。
   - 練習題目：列出**真正的題目/題單與來源**（考試制→考點題型+題庫；外商→Blind 75/NeetCode 具體題；民間→LeetCode 題單+實作專案），不要只寫「多練習」。
5. 在 `求職追蹤.md` 一覽表新增一列，並更新近期時程。

## 練習題結構（`練習題/`）

個人化練習題（依使用者背景：前端 React + 一點 Node）。每個軌道一個分類資料夾，含 `README.md` 題目目錄 + 每題一個子資料夾 `Q<NN>-<英文slug>-<中文>/`：

- **可執行程式題** → `題目.js`（題目註解 + stub + `node:assert` 測試，未解會 fail）+ `答案.js`（完整解 + 測試通過，印 `✅ 通過`）。測試用 CommonJS `require('node:assert')`，可直接 `node` 跑。
- **觀念/專案題**（計概、React/CSS、Node 專案）→ `題目.md` + `答案.md`。
- 新增題目：放進對應分類、依序編號、更新該分類 README。改完務必 `node 答案.js` 驗證通過。

## 內容原則

- 資訊以**官方公告為準**，文中適度加註免責提醒。
- 不複製有著作權的完整考古題；以代表性題型與可公開查證的資源為主。
- 數字/時程用**絕對日期**（YYYY-MM-DD），不要用「今年」「下個月」。

## 注意

- 目前日期感知：文件若提到考試時程，注意是否已過期，必要時提醒使用者更新。
