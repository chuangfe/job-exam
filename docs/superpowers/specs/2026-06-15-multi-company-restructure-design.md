# 多公司求職追蹤庫 — 重整設計

**日期**：2026-06-15
**目標**：將目前單一公司（中華電信）的考試準備庫，重整為可容納多種類型公司的求職追蹤庫，並實際產出每家公司的「學習方向」與「練習題目」內容。

---

## 1. 背景與問題

現況結構是為「單一公司」設計：

- 根目錄 `專案說明.md`、`目標職位.md` 內容**完全是中華電信專屬**，且與 `公司/中華電信/總覽.md` 高度重複。
- 資料夾結構（考試資訊、考古題）只適用「考試制」招募，無法套用民間企業與外商。
- `說明.md` 幾乎是空的（只有 `# job-exam`）。

要支援多公司、多類型（考試制 / 民間 / 外商），必須把「單一公司假設」拆掉。

## 2. 定位決策

**多公司求職追蹤庫**：所有公司平起平坐，中華電信只是其中一家。根目錄改為中立的索引與總覽，每家公司資料自給自足收在自己的資料夾。

## 3. 結構設計（方案 A：扁平 + 中立根目錄）

```
job-exam/
├── 說明.md                  # 擴寫為 README：專案介紹、結構、使用方式
├── 求職追蹤.md              # 新增：所有公司一覽表（類型/職位/時程/進度）
├── CLAUDE.md                # 新增：給未來 Claude 的維護規範
├── 範本/                    # 新增：三種類型的資料夾範本
│   ├── 考試制範本/
│   ├── 民間企業範本/
│   └── 外商範本/
└── 公司/
    ├── 中華電信/            # 既有（考試制），併入根目錄專屬內容
    ├── 國營聯招/            # 新增（考試制）
    ├── 中華郵政/            # 新增（考試制）
    ├── 台積電/             # 新增（民間）
    ├── Google/             # 新增（外商）
    └── Amazon/             # 新增（外商）
```

`專案說明.md`、`目標職位.md` 內容併入 `公司/中華電信/` 後，自根目錄移除（消除重複）。

## 4. 三種範本

共用骨架：**總覽 + 學習資料 + 面試準備**。

### 考試制範本（中華電信、國營聯招、中華郵政）
```
公司名/
├── 總覽.md
├── 考試資訊/
│   ├── 考試科目.md
│   ├── 報考資格與薪資.md
│   └── 招考時程.md
├── 學習資料/
│   ├── 學習方向.md      # 各科重點與讀書順序
│   ├── 練習題目.md      # 題型範例 + 推薦題單（考古題/Zerojudge/APCS）
│   └── 準備策略.md
├── 考古題/
│   └── 說明.md
└── 面試準備/
    └── 面試指南.md
```

### 民間企業範本（台積電）
```
公司名/
├── 總覽.md              # 含職缺 JD 與技術棧
├── 學習資料/
│   ├── 學習方向.md      # 對應 JD 的技術學習路線
│   └── 練習題目.md      # LeetCode 中等題 + 實作專案
├── 履歷準備.md
└── 面試準備/
    └── 面試指南.md      # 技術面 + HR 面
```

### 外商範本（Google、Amazon）
```
公司名/
├── 總覽.md              # 含面試流程（各關卡）
├── 學習資料/
│   ├── 學習方向.md      # DSA topics + 系統設計路線
│   └── 練習題目.md      # Blind 75 / NeetCode 150 題單
├── 履歷準備.md
└── 面試準備/
    ├── 技術面試.md      # coding + system design
    └── 行為面試.md      # STAR / Amazon Leadership Principles
```

## 5. 核心內容要求：學習方向 + 練習題目

每家公司都要有實質內容，不是空殼：

- **學習方向.md**：依該公司/職位列出學什麼、順序、深度重點。
- **練習題目.md**：**列出真正的題目清單與來源**
  - 考試制：高頻考點題型範例 + Zerojudge / APCS / 國營考古題題單
  - 外商：Blind 75 / NeetCode 150 精選題（按主題分類列題名）
  - 民間：LeetCode 中等題單 + 實作專案建議

## 6. CLAUDE.md 內容

給未來維護用：
- 專案性質（求職追蹤庫，繁體中文）
- 目錄結構與命名慣例
- **新增公司 SOP**：判斷類型 → 複製對應範本 → 要填哪些檔案 → 更新 `求職追蹤.md` 一覽表
- 資訊以官方公告為準的免責提醒

## 7. 範圍界線（YAGNI）

- 暫不做依類型分群的子資料夾（公司數量還少，扁平即可）。
- 不動既有中華電信的內容品質（僅做必要的根目錄內容併入）。
- 題單以具代表性、可公開查證的資源為主；不複製有著作權的完整考古題。
