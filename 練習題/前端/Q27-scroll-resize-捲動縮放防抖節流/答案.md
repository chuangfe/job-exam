# Q27 - scroll / resize 該用防抖還是節流？解答

## 結論

`scroll` / `resize` 每秒可觸發數十到上百次，若每次都做昂貴運算（讀佈局、重排版、發請求）會卡頓掉幀。選擇的判斷依據是：

- **需要「過程中持續反應」 → 用 throttle（節流）**：固定間隔執行一次，過程中仍有更新。
- **只需要「結束後處理一次」 → 用 debounce（防抖）**：停止觸發一段時間後才執行一次。

## 對照表

| 情境 | 選擇 | 原因 |
| --- | --- | --- |
| 捲動進度條 / 視差效果 | throttle | 過程中要持續更新畫面 |
| 無限捲動「是否接近底部」判斷 | throttle（或改用 IntersectionObserver） | 捲動中要持續檢查 |
| resize 後重新計算佈局 / 重繪圖表 | debounce | 只需在停止調整後算一次 |
| 送出 analytics（如記錄最終捲動深度） | debounce | 只在停止後送一次，省請求 |
| 搜尋框輸入即時建議 | debounce | 停止輸入才查詢 |

## 範例

```tsx
import { useEffect } from "react";

// throttle：捲動進度條，過程中持續更新
function useScrollProgress(onProgress: (p: number) => void) {
  useEffect(() => {
    let ticking = false;
    const handler = () => {
      if (ticking) return;
      ticking = true;
      // 用 rAF 做節流，對齊瀏覽器繪製節奏
      requestAnimationFrame(() => {
        const { scrollTop, scrollHeight, clientHeight } =
          document.documentElement;
        onProgress(scrollTop / (scrollHeight - clientHeight));
        ticking = false;
      });
    };
    window.addEventListener("scroll", handler, { passive: true });
    return () => window.removeEventListener("scroll", handler);
  }, [onProgress]);
}

// debounce：resize 結束後才重算佈局
function useDebouncedResize(onResize: () => void, delay = 200) {
  useEffect(() => {
    let timer: ReturnType<typeof setTimeout>;
    const handler = () => {
      clearTimeout(timer);
      timer = setTimeout(onResize, delay); // 停止 resize 後才執行一次
    };
    window.addEventListener("resize", handler);
    return () => {
      clearTimeout(timer);
      window.removeEventListener("resize", handler);
    };
  }, [onResize, delay]);
}
```

## 更現代的替代方案

很多場景根本不需要監聽 scroll / resize，改用 Observer 由瀏覽器在背景高效偵測，更省效能也更準確：

- **IntersectionObserver**：判斷元素是否進入視窗（無限捲動、懶載入、曝光追蹤）取代 scroll 監聽。
- **ResizeObserver**：監測「特定元素」尺寸變化（不只是視窗）取代 resize 監聽。

```tsx
// 用 IntersectionObserver 取代「監聽 scroll 判斷是否到底部」
const sentinelRef = useRef<HTMLDivElement>(null);
useEffect(() => {
  const io = new IntersectionObserver(([entry]) => {
    if (entry.isIntersecting) loadMore();
  });
  if (sentinelRef.current) io.observe(sentinelRef.current);
  return () => io.disconnect();
}, []);
```
