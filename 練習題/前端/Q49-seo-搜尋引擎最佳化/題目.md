# 前端 SEO（搜尋引擎最佳化）

請依序回答以下前端 SEO 核心問題。SEO 不是「行銷才管的事」——前端能直接掌控的部分（meta、語意化 HTML、渲染方式、效能、結構化資料）往往決定了頁面能不能被正確索引、能不能拿到 rich results。建議先自己用白話講出來，再對照解析。以 Next.js（App Router）為主要示範環境。

> 提醒：本題與 **Q29 核心網頁指標（Core Web Vitals）**、**Q37 Next.js 渲染策略** 互補，建議一起服用。

## Q1. 基本 meta 與語意化 HTML？

- `<title>` 與 `<meta name="description">` 各自的作用是什麼？會不會直接影響排名？在搜尋結果（SERP）上分別對應到哪裡？合理長度大約多少？
- `<meta name="viewport">` 跟 SEO 有什麼關係？（提示：行動裝置友善、mobile-first indexing）
- 什麼叫「語意化 HTML」？用 `<header>`/`<nav>`/`<main>`/`<article>`/`<footer>` 而不是一堆 `<div>`，對爬蟲有什麼好處？
- 標題階層（`<h1>`~`<h6>`）為什麼重要？一頁該有幾個 `<h1>`？亂跳階層（h1 直接跳 h3）有什麼問題？
- 圖片的 `alt` 屬性在 SEO 與無障礙上扮演什麼角色？

## Q2. Open Graph 與 Twitter Card？

- Open Graph 是什麼？它主要解決「哪個場景」的問題？（提示：社群分享預覽，不是 Google 排名）
- 列出幾個最關鍵的 og 標籤（`og:title`、`og:description`、`og:image`、`og:type`、`og:url`）並說明用途。
- `og:image` 有什麼實務要點（尺寸、絕對網址、檔案大小）？
- Twitter Card（`twitter:card`、`twitter:title`、`twitter:image`…）跟 Open Graph 是什麼關係？沒有 twitter 標籤時 Twitter/X 會怎麼做？
- 在 Next.js App Router 要怎麼產生這些標籤？

## Q3. Canonical URL？

- 什麼是「重複內容（duplicate content）」問題？舉幾個常見成因（帶 query string 的網址、`www` 與非 `www`、http/https、分頁、UTM 參數…）。
- `<link rel="canonical">` 怎麼解決這個問題？它是「命令」還是「建議」？
- 自我參照（self-referencing）canonical 是什麼、為什麼建議做？
- canonical 用相對路徑還是絕對路徑？

## Q4. 結構化資料 JSON-LD？

- 什麼是結構化資料（structured data）？Schema.org 與 JSON-LD 的關係是什麼？為什麼 Google 推薦用 JSON-LD 而非 microdata？
- 結構化資料能帶來什麼好處？（提示：rich results / 複合式搜尋結果——星等、麵包屑、FAQ 展開、商品價格…）
- 請說明以下幾種常見類型分別用在什麼頁面：`Article`、`Product`、`BreadcrumbList`、`FAQPage`。
- JSON-LD 放在頁面哪裡？結構化資料的內容一定要跟頁面上「使用者看得到」的內容一致嗎？（提示：不一致的後果）
- 怎麼驗證結構化資料寫得對不對？

## Q5. `sitemap.xml` 與 `robots.txt`？

- `sitemap.xml` 的作用是什麼？它是「保證被收錄」還是「幫助被發現」？裡面通常放哪些資訊（loc、lastmod…）？
- `robots.txt` 的作用是什麼？`Disallow` 是「禁止索引」還是「禁止爬取」？兩者有什麼陷阱？（提示：被 disallow 的頁面仍可能因外部連結被索引，只是內容抓不到）
- 「不想被索引某一頁」正確做法是什麼？（提示：`<meta name="robots" content="noindex">` vs robots.txt 的差異）
- 在 Next.js App Router 要怎麼用程式產生 `sitemap.xml` 與 `robots.txt`？

## Q6. 渲染方式與 SEO？

- CSR（純前端渲染，例如傳統 CRA / Vite SPA）對爬蟲有什麼風險？「首包 HTML 幾乎是空的、靠 JS 補內容」會怎樣？
- Google 說它會執行 JS（第二波索引 / two-wave indexing），那為什麼還是建議 SSR/SSG？（提示：延遲、預算、其他非 Google 爬蟲如社群分享爬蟲不跑 JS）
- SSR / SSG 為什麼對 SEO 有利？
- 什麼是 hydration？「內容已在 HTML 裡，但互動要等 JS」對 SEO 與使用者體驗分別有什麼影響？
- 在 Next.js App Router 用 **Metadata API** 管理 SEO 有哪些方式？靜態 `metadata` 與動態 `generateMetadata` 各用在什麼情境？
