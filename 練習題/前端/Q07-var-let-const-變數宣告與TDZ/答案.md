# var / let / const 與 TDZ 解答

## 差異比較表

| 特性 | var | let | const |
|------|-----|-----|-------|
| 作用域 | 函式 | 區塊 | 區塊 |
| 重複宣告 | 允許 | 不允許 | 不允許 |
| 提升後可用 | 是（undefined） | 否（TDZ） | 否（TDZ） |
| 必須初始化 | 否 | 否 | 是 |
| 重新賦值 | 可 | 可 | 不可（但物件內容可變） |

## TDZ（暫時死區）

TDZ 是從區塊開頭到宣告語句之間的範圍，`let` / `const` 在此區間內被存取會報錯，目的是強制「先宣告再使用」。

## const 的本質

`const` 保證的是「綁定不變」，不是「值不可變」；要凍結內容用 `Object.freeze`（淺凍結）。

```js
const obj = { a: 1, nested: { b: 2 } };
obj.a = 10;            // OK，內容可變
// obj = {};           // TypeError，綁定不可變

Object.freeze(obj);
obj.a = 999;           // 靜默失敗（嚴格模式下報錯）
obj.nested.b = 999;    // 仍可變！Object.freeze 是淺凍結
```
