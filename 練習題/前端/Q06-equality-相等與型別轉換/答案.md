# == vs === 與型別轉換解答

## 重點整理

- `===` 嚴格相等，型別不同直接 `false`，不做轉換。
- `==` 寬鬆相等會做型別轉換，規則複雜，面試常考結果：
  - `null == undefined` → `true`；兩者與其他值比較皆 `false`（除彼此之外）。
  - 數字與字串比較：字串轉數字。
  - 布林參與比較：布林先轉數字。
  - 物件與原始型別比較：物件先 ToPrimitive（`valueOf` / `toString`）。
- `NaN === NaN` 為 `false`，要用 `Number.isNaN` 判斷。
- 實務建議：一律用 `===`；需判 `null` / `undefined` 時用 `x == null`。

## 程式碼示範

```js
0 == '';         // true（'' → 0）
0 == '0';        // true
'' == '0';       // false（字串比字串，不轉換）
null == 0;       // false（null 只等於 undefined）
[] == ![];       // true（![]→false→0，[]→''→0）
NaN == NaN;      // false
```

特別注意 `[] == ![]`：`![]` 先把 `[]` 轉成布林 `true` 再取反得 `false`，`false` 再轉數字為 `0`；左邊 `[]` 經 ToPrimitive 得空字串 `''`，再轉數字也是 `0`，因此兩邊相等。
