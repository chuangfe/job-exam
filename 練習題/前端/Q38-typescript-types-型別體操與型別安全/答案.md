# TypeScript 型別體操與型別安全 解析

## Q1. 泛型、泛型約束與預設型別參數

**泛型解決什麼？** 讓型別「參數化」，寫一次邏輯就能保留「輸入型別 → 輸出型別」的關聯。跟 `any` 最大的差別是：`any` 把型別資訊丟掉了，泛型把它「記住並傳遞下去」。

```ts
// any：丟失型別，回傳也是 any，之後怎麼用都不報錯（危險）
function identityAny(x: any): any { return x; }
const a = identityAny(123); // a: any

// 泛型：呼叫時 T 被推論成 number，關聯保留下來
function identity<T>(x: T): T { return x; }
const b = identity(123); // b: number
```

**泛型約束 `extends`**：限制型別參數必須「相容於」某個型別，這樣才能安全地存取它的成員。經典題是型別安全的 `getProp`：

```ts
// K 只能是 T 的 key，取出的值型別自動是 T[K]
function getProp<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = { id: 1, name: 'Ian' };
getProp(user, 'name'); // string
getProp(user, 'id');   // number
// getProp(user, 'age'); // ❌ 編譯錯：'age' 不是 keyof user
```

**預設型別參數**：型別參數也能有預設值，呼叫端沒指定時用預設。

```ts
type ApiResult<T = unknown> = { ok: boolean; data: T };
type R1 = ApiResult;          // data: unknown
type R2 = ApiResult<string>;  // data: string
```

白話：`extends` 是「約束上限」，讓 TS 敢放你存取成員；`= 型別` 是「預設下限」，沒填就用它。

## Q2. 內建 Utility Types

| 型別 | 用途（一句話） |
|------|----------------|
| `Partial<T>` | 所有屬性變可選 |
| `Required<T>` | 所有屬性變必填 |
| `Readonly<T>` | 所有屬性變唯讀 |
| `Pick<T, K>` | 只留下 K 這些 key |
| `Omit<T, K>` | 移除 K 這些 key |
| `Record<K, V>` | 建立「key 為 K、值為 V」的物件型別 |
| `ReturnType<F>` | 取函式回傳型別 |
| `Parameters<F>` | 取函式參數型別（tuple） |
| `Awaited<T>` | 取 Promise resolve 後的型別（會遞迴拆） |
| `NonNullable<T>` | 從 T 移除 `null` 與 `undefined` |

```ts
interface User { id: number; name: string; email?: string }

type A = Partial<User>;   // { id?: number; name?: string; email?: string }
type B = Required<User>;  // { id: number; name: string; email: string }
type C = Readonly<User>;  // { readonly id: number; ... }

type D = Pick<User, 'id' | 'name'>; // { id: number; name: string }
type E = Omit<User, 'email'>;       // { id: number; name: string }

type Roles = 'admin' | 'guest';
type F = Record<Roles, boolean>;    // { admin: boolean; guest: boolean }

function fn(a: number, b: string) { return { a, b }; }
type G = ReturnType<typeof fn>;     // { a: number; b: string }
type H = Parameters<typeof fn>;     // [a: number, b: string]

type I = Awaited<Promise<Promise<number>>>; // number
type J = NonNullable<string | null | undefined>; // string
```

白話：`Pick`/`Omit` 是一體兩面（要哪些 vs 不要哪些）；`ReturnType`/`Parameters` 常拿來「延用既有函式的型別」，避免重複宣告。

## Q3. 條件型別、`infer` 與分布式條件型別

**`T extends U ? X : Y`**：如果 `T` 可以賦值給 `U`，型別就是 `X`，否則是 `Y`。它是「型別層級的三元運算子」。

**`infer`**：在 `extends` 子句裡「宣告一個型別變數，讓 TS 幫你反推填入」。取陣列元素型別：

```ts
type ElementType<T> = T extends (infer E)[] ? E : never;

type X1 = ElementType<string[]>; // string
type X2 = ElementType<number[]>; // number
type X3 = ElementType<boolean>;  // never（不是陣列）
```

白話：`infer E` 是跟 TS 說「這個位置我不知道是啥，你比對時抓到什麼就存進 E」。

**分布式條件型別**：當條件型別作用在「裸的型別參數」而該參數是聯合型別時，會「拆開逐一套用再合併」：

```ts
type ToArray<T> = T extends any ? T[] : never;
type Y1 = ToArray<string | number>;
// 分配成 ToArray<string> | ToArray<number>
// = string[] | number[]  （不是 (string | number)[]）
```

想「關掉分配」，把兩邊都用 tuple 包起來 `[T] extends [U]`：

```ts
type ToArrayNonDist<T> = [T] extends [any] ? T[] : never;
type Y2 = ToArrayNonDist<string | number>; // (string | number)[]
```

實務常見用它實作 `MyExclude`：

```ts
type MyExclude<T, U> = T extends U ? never : T;
type Z = MyExclude<'a' | 'b' | 'c', 'b'>; // 'a' | 'c'
```

## Q4. 判別聯合與型別窄化

**判別聯合**：一組物件型別，靠一個共同的「字面量欄位」（判別鍵，如 `type`/`status`）區分。TS 一旦比對到那個欄位的值，就能把型別窄化到對應成員。很適合「API 狀態」「Redux action」這種「同一件事有多種互斥形態」的資料。

```ts
type Result =
  | { status: 'loading' }
  | { status: 'success'; data: string }
  | { status: 'error'; message: string };

function render(r: Result) {
  switch (r.status) {
    case 'loading': return 'Loading...';
    case 'success': return r.data;      // 這裡 TS 知道有 data
    case 'error':   return r.message;   // 這裡 TS 知道有 message
  }
}
```

**四種窄化手段**：

```ts
// 1) typeof：窄化「原始型別」(string/number/boolean/...)
function f1(x: string | number) {
  if (typeof x === 'string') x.toUpperCase(); // x: string
}

// 2) in：靠「屬性是否存在」窄化物件聯合
type A = { swim: () => void };
type B = { fly: () => void };
function f2(x: A | B) {
  if ('swim' in x) x.swim(); // x: A
  else x.fly();              // x: B
}

// 3) instanceof：窄化「class 實例」
function f3(x: Date | string) {
  if (x instanceof Date) x.getTime(); // x: Date
}

// 4) 自訂 type guard：回傳 `x is T`，把複雜判斷封裝成可重用的守衛
type Cat = { meow: () => void };
function isCat(x: unknown): x is Cat {
  return typeof x === 'object' && x !== null && 'meow' in x;
}
function f4(x: unknown) {
  if (isCat(x)) x.meow(); // x: Cat
}
```

白話：`typeof` 管原始型別、`instanceof` 管 class、`in` 管「有沒有這個屬性」、`x is T` 是自己寫的守衛函式（回傳 boolean，但同時告訴 TS 窄化結果）。

**窮盡檢查（exhaustiveness）**：在 `switch` 的 `default` 把值賦給 `never`，若未來新增了一種 case 卻忘了處理，型別就不會是 `never`，編譯期立刻報錯：

```ts
function render2(r: Result): string {
  switch (r.status) {
    case 'loading': return 'Loading...';
    case 'success': return r.data;
    case 'error':   return r.message;
    default: {
      const _exhaustive: never = r; // 少處理一種就編譯錯
      return _exhaustive;
    }
  }
}
```

## Q5. `as const`、字面量型別與 `satisfies`

**字面量型別**：型別本身就是某個「具體值」，如 `'admin'`、`42`、`true`。`const` 宣告的原始值會被推論成「字面量型別」，`let` 則被「放寬（widen）」成一般型別：

```ts
const x = 'hello'; // 型別：'hello'（字面量）
let y = 'hello';   // 型別：string（放寬）
```

**`as const`**：把整個字面量「凍結」成最窄、且 `readonly` 的型別。物件的屬性變唯讀、陣列變 readonly tuple：

```ts
const config = { mode: 'dark', size: 10 } as const;
// 型別：{ readonly mode: 'dark'; readonly size: 10 }

const arr = [1, 2, 3] as const;
// 型別：readonly [1, 2, 3]

// 常用來從陣列產生聯合型別：
const ROLES = ['admin', 'guest'] as const;
type Role = typeof ROLES[number]; // 'admin' | 'guest'
```

**`satisfies`**：檢查一個值「符合」某型別，但「不放寬」它的推論型別。解決「想要型別檢查、又想保留精確字面量」的兩難：

```ts
type Config = Record<string, string | number>;

// 用 : Config → 通過檢查，但取值時型別被放寬
const c1: Config = { mode: 'dark', size: 10 };
c1.mode; // string | number（丟失了 'dark'）

// 用 satisfies → 一樣做檢查，但保留精確型別
const c2 = { mode: 'dark', size: 10 } satisfies Config;
c2.mode; // string（保留精確）
c2.size; // number
// 且拼錯或型別不符仍會被抓：{ mode: true } satisfies Config → ❌
```

三者差別：
- `: T`（型別註記）：做檢查，但把值「當成 T」，會放寬、丟失細節。
- `as T`（斷言）：**不做安全檢查**，硬把值當成 T，用錯會出事。
- `satisfies T`：做檢查，但「保留原本推論的精確型別」——最推薦。

## Q6. `unknown` vs `any` vs `never`

- **`any`**：關掉型別檢查。可以賦值給任何型別、也接受任何操作 → 方便但不安全，等於放棄 TS 的保護。
- **`unknown`**：「型別安全版的 any」。任何值都能賦給 `unknown`，但你**不能直接對它做任何操作**，必須先窄化。
- **`never`**：「不可能存在的值」。沒有任何值能賦給 `never`（除了 never 自己）；它可賦值給任何型別。常見於「不會回傳的函式」「窮盡檢查」「條件型別的 fallback」。

用「賦值方向」記：

```ts
// unknown：什麼都能塞進來，但塞出去要先窄化
let u: unknown = 123;
// u.toFixed();        // ❌ 不准直接用
if (typeof u === 'number') u.toFixed(); // ✅ 窄化後才行

// any：什麼都能塞、什麼都能做（不報錯 = 不安全）
let a: any = 123;
a.foo.bar.baz(); // 編譯通過，但執行時炸

// never：沒有值能塞進 never；never 可以塞給任何型別
function fail(msg: string): never { throw new Error(msg); }
```

**為什麼用 `unknown` 取代 `any`**：`unknown` 強迫你「先確認型別再使用」，把潛在錯誤擋在編譯期。典型例子是 `JSON.parse` / catch 的 error：

```ts
function parse(json: string): unknown {
  return JSON.parse(json);
}
const data = parse('{"name":"Ian"}');
// data.name;            // ❌ unknown 不准直接存取
if (typeof data === 'object' && data !== null && 'name' in data) {
  // 這裡才安全存取
}

try { /* ... */ } catch (e) {
  // e 預設是 unknown，強迫你先判斷
  if (e instanceof Error) console.log(e.message);
}
```

`never` 常見情境：`throw`/無窮迴圈的回傳、窮盡檢查、被過濾到空的聯合型別（`Exclude<'a','a'>` → `never`）。

## Q7.（挑戰）手刻 Utility Types

```ts
// (a) MyPick：用 mapped type 走訪 K，值取 T[P]
type MyPick<T, K extends keyof T> = {
  [P in K]: T[P];
};
type R1 = MyPick<{ a: 1; b: 2; c: 3 }, 'a' | 'c'>; // { a: 1; c: 3 } ✅

// (b) MyPartial：加上 `?` 修飾符
type MyPartial<T> = {
  [P in keyof T]?: T[P];
};
type R2 = MyPartial<{ a: 1; b: 2 }>; // { a?: 1; b?: 2 } ✅

// (c) MyReadonly：加上 `readonly` 修飾符
type MyReadonly<T> = {
  readonly [P in keyof T]: T[P];
};
type R3 = MyReadonly<{ a: 1 }>; // { readonly a: 1 } ✅

// (d) DeepReadonly：巢狀遞迴——值若是物件就再套一層
type DeepReadonly<T> = {
  readonly [P in keyof T]: T[P] extends object ? DeepReadonly<T[P]> : T[P];
};
type R4 = DeepReadonly<{ a: { b: { c: 1 } } }>;
// { readonly a: { readonly b: { readonly c: 1 } } } ✅

// (e) MyReturnType：用 infer 抓回傳型別
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never;
type R5 = MyReturnType<() => string>;              // string ✅
type R6 = MyReturnType<(x: number) => { ok: boolean }>; // { ok: boolean } ✅
```

白話重點：
- **Mapped type** `{ [P in K]: ... }` 是「型別層級的 for 迴圈」，逐一走訪 key。
- 修飾符 `?`（可選）與 `readonly`（唯讀）直接加在 mapped type 上就能開關；要移除則用 `-?` / `-readonly`。
- **`DeepReadonly`** 靠「條件型別 + 遞迴」：值是 `object` 就再對它套一次 `DeepReadonly`，否則保留原值。（嚴謹版還要處理陣列/函式等，這裡示範核心概念。）
- **`infer R`** 讓 TS 從函式簽章「反推」出回傳型別填進 `R`。

## Q8. React 情境的型別

**元件 props**：

```tsx
import { ReactNode, MouseEvent } from 'react';

type ButtonProps = {
  variant?: 'primary' | 'secondary'; // 選填聯合字面量
  onClick: () => void;               // 必填
  children?: ReactNode;              // 可放任何可渲染內容
};

function Button({ variant = 'primary', onClick, children }: ButtonProps) {
  return <button className={variant} onClick={onClick}>{children}</button>;
}
```

**`useState` 泛型**：初始值是 `null` 時 TS 會把型別推成 `null`，之後放物件會噴錯。用泛型明確給出聯合型別：

```tsx
// ❌ 型別被推成 null，之後 setUser({...}) 會報錯
const [u1, setU1] = useState(null);

// ✅ 明確標注
const [user, setUser] = useState<User | null>(null);
```

**事件型別**：用 React 提供的泛型事件型別，並帶上元素型別：

```tsx
const onChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  console.log(e.target.value);
};
const onClick = (e: React.MouseEvent<HTMLButtonElement>) => {
  e.preventDefault();
};
const onSubmit = (e: React.FormEvent<HTMLFormElement>) => {
  e.preventDefault();
};
```

**`useRef` 兩種用法**：

```tsx
// 1) 指向 DOM：初始 null，型別給 DOM 節點，交給 ref 屬性用（唯讀 current）
const inputRef = useRef<HTMLInputElement>(null);
// inputRef.current?.focus();

// 2) 存可變值（不觸發重渲染）：給初始值，current 可寫
const timerRef = useRef<number>(0);
// timerRef.current = window.setTimeout(...);
```

差別：DOM ref 通常寫 `useRef<T>(null)`（TS 會把 current 視為唯讀的 `T | null`，交由 React 賦值）；可變值 ref 給非 null 初始值，`current` 就是可讀寫的一般欄位。

**泛型元件**（呼叫時自動推論 `T`）：

```tsx
type ListProps<T> = {
  items: T[];
  renderItem: (item: T) => ReactNode;
};

function List<T>({ items, renderItem }: ListProps<T>) {
  return <ul>{items.map((item, i) => <li key={i}>{renderItem(item)}</li>)}</ul>;
}

// 使用：T 被推論成 { id: number; name: string }
<List
  items={[{ id: 1, name: 'Ian' }]}
  renderItem={(u) => u.name} // u 已自動有型別
/>;
```

**`ComponentProps<typeof X>`**：取得「某個元件或原生標籤」的 props 型別，用來「延用/擴充」既有 props，不必自己重打一遍：

```tsx
import { ComponentProps } from 'react';

// 延用原生 <button> 的所有 props，再加自己的
type MyButtonProps = ComponentProps<'button'> & {
  variant?: 'primary' | 'secondary';
};

// 也能延用別人的元件 props
type ButtonProps2 = ComponentProps<typeof Button>;
```

白話：`ComponentProps<'button'>` = 原生 button 能接受的所有屬性（`onClick`、`disabled`、`type`...）；`ComponentProps<typeof Button>` = 你自訂元件 `Button` 的 props。做「包裝元件（wrapper）」時特別好用。
