# BFC（Block Formatting Context，區塊格式化情境）解答

## 什麼是 BFC
BFC 是一個**獨立的佈局區域**：在這個區域內部的元素佈局，不會影響到區域外部的元素（反之亦然）。它像是一個自我封閉的「容器」，內部自成一個排版環境。

## 如何觸發 BFC
常見的觸發條件：
- `overflow: hidden` / `auto`（非 `visible`）
- `display: flow-root`（**最推薦**，專為觸發 BFC 而生、無副作用）
- 浮動元素（`float: left` / `right`）
- 絕對定位（`position: absolute` / `fixed`）
- flex / grid 容器（其子項建立的格式化情境也具相似隔離效果）

```css
/* 最乾淨的觸發方式，無副作用 */
.container { display: flow-root; }

/* 傳統做法，但會裁切溢出內容 */
.container { overflow: hidden; }
```

## 用途
1. **清除浮動（包住浮動子元素）**：父元素建立 BFC 後，會把內部浮動子元素的高度算進來，解決高度塌陷。
   ```css
   .parent { display: flow-root; } /* 高度可正確包住內部 float 子元素 */
   .child  { float: left; }
   ```
2. **防止 margin 重疊（margin collapse）**：將其中一個元素包進獨立 BFC，可避免相鄰區塊的上下 margin 互相合併。
   ```css
   .wrapper { display: flow-root; } /* 內部元素的 margin 不再與外部重疊 */
   ```
3. **阻止文字環繞浮動元素**：讓內容區建立 BFC，它就不會與浮動元素重疊，文字會乖乖排在浮動元素旁的獨立欄位。
   ```css
   .float-box { float: left; width: 120px; }
   .content   { overflow: hidden; } /* 建立 BFC，內容不環繞浮動盒 */
   ```
