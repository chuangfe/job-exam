# 淺拷貝 vs 深拷貝解答

## 重點整理

- **淺拷貝**：只複製第一層，巢狀物件仍共用參考。常見做法：`{...obj}`、`Object.assign`、`Array.prototype.slice` / `concat`。
- **深拷貝**：遞迴複製所有層級，互不影響。

## 深拷貝方案比較

- `structuredClone(obj)`：現代瀏覽器原生支援，支援循環引用、`Map` / `Set` / `Date`，但不支援函式。
- `JSON.parse(JSON.stringify(obj))`：簡單，但會丟失 `undefined` / 函式 / `Symbol`，`Date` 會變字串，無法處理循環引用，`NaN` 會變成 `null`。
- 手寫遞迴：彈性最高，可自行處理特殊型別與循環引用。

## 程式碼示範

```js
const src = { a: 1, nested: { b: 2 } };

// 淺拷貝：nested 仍共用參考
const shallow = { ...src };
shallow.nested.b = 999;
console.log(src.nested.b); // 999（被影響）

// 深拷貝（structuredClone）
const deep = structuredClone(src);
deep.nested.b = 0;
console.log(src.nested.b); // 不受影響

// JSON 方式的限制
const weird = { d: new Date(), u: undefined, fn: () => {}, n: NaN };
console.log(JSON.parse(JSON.stringify(weird)));
// { d: "...字串...", n: null }（u 與 fn 直接消失）
```
