# Flexbox vs Grid（彈性與網格佈局）解答

## 根本差異
- **Flexbox（flex）**：**一維（單軸）**佈局。一次只專注於一個方向（水平 row 或垂直 column）的排列，元素會沿著主軸依內容自適應分配空間。
- **Grid**：**二維**佈局。可以**同時**定義「行（row）」與「列（column）」，適合整體版面的網格切分。

## 各自適合的場景
- **flex 適合**：
  - 一列導覽列、一排按鈕、工具列。
  - 元件內部依內容自適應排列（例如圖示 + 文字並排）。
  - 需要沿單一軸做對齊、分配剩餘空間（`justify-content`、`flex-grow`）。
- **grid 適合**：
  - 整體頁面版面（header / sidebar / main / footer）。
  - 卡片網格、相簿、表格狀的規律排版。
  - 需要同時控制行高與列寬、做精準對齊的場合。

## 程式碼示範
```css
/* flex：一排按鈕，水平排列、依內容自適應、彼此間隔 */
.toolbar {
  display: flex;
  gap: 8px;
  align-items: center;
}

/* grid：卡片網格，每欄最小 200px、自動填滿可用空間 */
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 16px;
}
```

## 實務搭配
常見做法是「外層用 grid 切版面、內部元件用 flex 排內容」：
```css
/* 外層 grid 負責整體版面 */
.layout {
  display: grid;
  grid-template-columns: 240px 1fr;
}

/* 每張卡片內部用 flex 排列內容 */
.card {
  display: flex;
  flex-direction: column;
  justify-content: space-between;
}
```
