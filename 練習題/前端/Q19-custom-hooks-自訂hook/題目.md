# 自訂 Hook 實作

請用 TypeScript 實作以下三個自訂 hook。先想清楚每個 hook 的時序與邊界情況，再動手。

## 1. `useDebounce<T>(value: T, delay?: number): T`
- 回傳「延遲後」的值：當 `value` 持續變動時，只有在停止變動超過 `delay` 毫秒後，回傳值才更新。
- 提示：用 `setTimeout` + `useEffect` 的清理函式，在值或 delay 變更時清掉舊計時器。

## 2. `usePrevious<T>(value: T): T | undefined`
- 取得「前一次渲染」的值（第一次渲染回傳 `undefined`）。
- 提示：思考 `useEffect` 的執行時機——它在渲染後才執行，能否利用這個時序差，把值存進 ref？

## 3. `useFetch<T>(url: string): { data, loading, error }`
- 依 `url` 抓資料，回傳 `data` / `loading` / `error` 三種狀態。
- 需求重點：
  - 切換 `url` 時要能取消前一個請求（提示：`AbortController`）；
  - 避免競態（race condition）——快速切換 url 時，不可讓較早回來的舊回應覆蓋較新的結果；
  - 元件卸載後不可再 `setState`（避免警告與記憶體洩漏）；
  - `AbortError` 不應被當成錯誤呈現。
