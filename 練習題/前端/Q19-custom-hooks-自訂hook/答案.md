# 自訂 Hook 解析

## 完整程式碼

```tsx
import { useState, useEffect, useRef, useCallback } from 'react';

// useDebounce：回傳延遲後的值
function useDebounce<T>(value: T, delay = 300): T {
  const [debounced, setDebounced] = useState(value);
  useEffect(() => {
    const id = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(id); // 值或 delay 變更時清掉舊計時器
  }, [value, delay]);
  return debounced;
}

// usePrevious：取得前一次渲染的值
function usePrevious<T>(value: T): T | undefined {
  const ref = useRef<T>();
  useEffect(() => { ref.current = value; }, [value]);
  return ref.current; // effect 在渲染後才更新，故回傳的是上一輪的值
}

// useFetch：含 loading/error、abort 取消、避免競態
interface FetchState<T> { data: T | null; loading: boolean; error: Error | null; }

function useFetch<T>(url: string): FetchState<T> {
  const [state, setState] = useState<FetchState<T>>({
    data: null, loading: true, error: null,
  });
  useEffect(() => {
    const controller = new AbortController();
    setState({ data: null, loading: true, error: null });
    fetch(url, { signal: controller.signal })
      .then((res) => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        return res.json();
      })
      .then((data: T) => setState({ data, loading: false, error: null }))
      .catch((error: Error) => {
        if (error.name === 'AbortError') return; // 取消不算錯誤
        setState({ data: null, loading: false, error });
      });
    return () => controller.abort(); // url 變更或卸載時取消，避免競態與記憶體洩漏
  }, [url]);
  return state;
}
```

## 重點解析

### useDebounce
- 每次 `value` / `delay` 變更都重設一個計時器；清理函式會在下一次 effect 執行前清掉上一個計時器。
- 因此只有「停止變動 `delay` 毫秒」後，`setDebounced` 才真的觸發，達到防抖效果。

### usePrevious
- 利用「effect 在渲染後執行」的時序：渲染階段先讀取 `ref.current`（仍是舊值並回傳），渲染後 effect 才把 `ref.current` 更新為當前值。
- 所以本次渲染拿到的永遠是上一輪的值；第一次渲染時 ref 尚未被寫入，回傳 `undefined`。

### useFetch
- 用 `AbortController` 解決快速切換造成的競態：`url` 變更時清理函式呼叫 `controller.abort()`，舊請求被取消，較早回來的舊回應不會覆蓋新結果。
- 同一個清理機制也處理「卸載後 setState」警告——卸載時 effect 清理觸發 abort，被取消的 promise 進到 `catch` 後因 `AbortError` 直接 return，不會再呼叫 `setState`。

### 關於 `useCallback`
- 本範例 `import` 了 `useCallback` 但未實際使用，程式碼保留與來源一致即可。
- 實務上若要把回呼（如重新抓取的 `refetch` 函式）回傳給外部使用，可用 `useCallback` 包裝以穩定其參考，避免外部 effect 反覆觸發。
