# 修正 useEffect 無限迴圈 解析

## 問題分析

```tsx
// 錯誤版：每次渲染 setState → 重渲染 → 再次執行 effect → 無限迴圈
function Buggy() {
  const [data, setData] = useState({ count: 0 });
  useEffect(() => {
    setData({ count: data.count + 1 }); // 沒有依賴控制 + 每次都 setState
  });                                     // 缺依賴陣列，每次渲染都跑
  return <div>{data.count}</div>;
}
```

兩個原因疊加造成迴圈：
1. **缺依賴陣列** → effect 在「每次渲染後」都會執行。
2. **effect 內無條件 `setState`** → 觸發新一輪渲染 → 渲染後又執行 effect → 再 setState → 循環不止。

## 修正版

```tsx
// 修正：明確依賴 + 用函式式更新或加入終止條件
function Fixed() {
  const [count, setCount] = useState(0);
  useEffect(() => {
    if (count < 5) {
      const id = setTimeout(() => setCount((c) => c + 1), 1000);
      return () => clearTimeout(id);
    }
  }, [count]); // 明確依賴，且有終止條件 count < 5
  return <div>{count}</div>;
}
```

## 重點

無限迴圈常見來源：
- 缺依賴陣列；
- 依賴中放每次渲染都新建的物件/陣列/函式（參考每次都變，等同沒做記憶）；
- effect 內無條件更新自己依賴的 state。

修法：
- 補上依賴陣列；
- 用 `useCallback` / `useMemo` 穩定參考；
- 用函式式更新並設終止條件（如本例 `count < 5`），並記得清理計時器。
