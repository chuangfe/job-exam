# Q45 - 網頁可及性 a11y（Accessibility / WCAG）

## 問題

「可及性（Accessibility，常縮寫為 a11y）」是指讓所有人——包含視障、聽障、行動不便、認知障礙，以及只用鍵盤或使用螢幕報讀器的使用者——都能正常使用網站。本題請你說明可及性的重要性、核心原則，並實際重構一段「不可及」的元件。

## 引導思考

### Q1. 為何重要與 WCAG 四原則

- 為什麼前端要重視可及性？請從**法規責任**、**使用者涵蓋範圍**、**SEO 與商業價值**三個角度說明。
- WCAG（Web Content Accessibility Guidelines）的四大原則「POUR」分別是什麼？各代表什麼意思？
- WCAG 有 A / AA / AAA 三個等級，業界通常以哪一級為目標？

### Q2. 語意化 HTML 與 ARIA 的關係

- 為什麼「語意化 HTML」是可及性的地基？用 `<div onClick>` 當按鈕會少了哪些原生能力？
- 「ARIA 第一守則（No ARIA is better than bad ARIA）」是什麼意思？什麼時候才真的需要 ARIA？
- 舉例說明：哪些情況原生 HTML 就夠了、哪些情況才需要補 ARIA。

### Q3. ARIA 的三類屬性

- ARIA 屬性大致分成 role、state、property 三類，各是什麼？
- 說明以下屬性的用途與差異：
  - `aria-label` / `aria-labelledby` / `aria-describedby`
  - `aria-expanded` / `aria-hidden`
  - `aria-live`

### Q4. 鍵盤操作與焦點管理

- 什麼是 tab 順序？`tabindex` 的 `0`、`-1`、正整數各代表什麼？為什麼不建議用正整數？
- 為什麼「絕對不要移除 focus 樣式（`outline: none`）」？若要自訂又該怎麼做？
- 什麼是「focus 管理」與「focus trap（焦點陷阱）」？在 Modal / Dialog 開關時應如何處理焦點？

### Q5. 螢幕報讀器與動態通知

- 螢幕報讀器（screen reader）大致如何運作？它讀的是畫面還是什麼？
- 表單送出後出現的錯誤訊息、或右下角跳出的 toast，為什麼報讀器可能「讀不到」？`aria-live` 的 `polite` 與 `assertive` 差在哪？

### Q6. 色彩、表單與其他常見要點

- WCAG AA 對「一般文字」的色彩對比要求是多少？為什麼「不能只用顏色傳達資訊」？
- 說明以下各自的可及性要點：可及的表單（label 綁定、錯誤提示）、按鈕 vs 連結該怎麼選、圖片 `alt`（含裝飾性圖片）、跳過導覽連結（skip link）。

### Q7. 測試工具

- 列出至少四種檢測可及性的方式，並說明各自能抓到 / 抓不到什麼（自動化工具無法取代人工的原因）。

### Q8. 重構實作

- 給定下面兩段「不可及」的程式碼，請重構成可及版本，並逐項說明改了什麼、為什麼。

**（A）用 div 拼出來的 Modal：**

```tsx
function BadModal({ open, onClose }) {
  if (!open) return null;
  return (
    <div className="overlay" onClick={onClose}>
      <div className="modal">
        <div className="title">確認刪除</div>
        <div>此操作無法復原，確定要刪除嗎？</div>
        <div className="btn" onClick={onClose}>取消</div>
        <div className="btn" onClick={onClose}>刪除</div>
        <div className="close" onClick={onClose}>✕</div>
      </div>
    </div>
  );
}
```

**（B）用 div 拼出來的自訂下拉選單：**

```tsx
function BadSelect({ options, value, onChange }) {
  const [open, setOpen] = useState(false);
  return (
    <div className="select">
      <div className="selected" onClick={() => setOpen(!open)}>
        {value}
      </div>
      {open && (
        <div className="options">
          {options.map((opt) => (
            <div key={opt} className="option" onClick={() => onChange(opt)}>
              {opt}
            </div>
          ))}
        </div>
      )}
    </div>
  );
}
```
