# this 綁定解答

## 綁定規則（優先序由高到低）

1. **new 綁定**：`this` 指向新建立的物件。
2. **顯式綁定**：`call` / `apply` / `bind` 指定的物件。
3. **隱式綁定**：`obj.fn()`，`this` 為呼叫點前的物件。
4. **預設綁定**：嚴格模式為 `undefined`，非嚴格模式為全域物件。

## 箭頭函式

箭頭函式沒有自己的 `this`，會沿用「定義時」外層作用域的 `this`，而且無法被 `call` / `apply` / `bind` 改變。

## call / apply / bind 差異

| 方法 | 是否立即執行 | 傳參方式 |
|------|------------|---------|
| `call`  | 是 | 逐一傳入 `fn.call(ctx, a, b)` |
| `apply` | 是 | 陣列傳入 `fn.apply(ctx, [a, b])` |
| `bind`  | 否（回傳新函式） | 逐一傳入，可分次（柯里化） |

## 程式碼示範

```js
const obj = { x: 1 };
function f(a, b) { return this.x + a + b; }
f.call(obj, 2, 3);       // 6
f.apply(obj, [2, 3]);    // 6
const g = f.bind(obj, 2);
g(3);                    // 6（後綁定的 this 無效，bind 的 this 不可再改）
```

`bind` 回傳一個新函式，且綁定的 `this` 之後無法再被覆寫；`bind(obj, 2)` 同時預先帶入第一個參數 `2`，這就是柯里化的應用。
