# Q25 - CORS（跨來源資源共享）

## 問題

CORS（Cross-Origin Resource Sharing，跨來源資源共享）是什麼？它要解決什麼問題？「預檢請求（preflight）」在什麼情況下會觸發？跨來源請求要帶 cookie 時，前後端各要設定什麼？

## 引導思考

- 「同源（same-origin）」的定義是什麼？協定、網域、埠號之中改變哪一個就算跨源？
- 同源政策（Same-Origin Policy）限制了什麼？CORS 是在「放寬」還是「加強」這個限制？
- 什麼是「簡單請求（simple request）」？它和「非簡單請求」的差別在哪？
- 預檢請求用的是哪個 HTTP 方法？伺服器要回哪些標頭才會放行？
- 哪些情況（HTTP 方法、自訂標頭、`Content-Type`）會讓請求變成「非簡單」而需要預檢？
- 要帶 cookie 跨源時，為什麼 `Access-Control-Allow-Origin` 不能是 `*`？
