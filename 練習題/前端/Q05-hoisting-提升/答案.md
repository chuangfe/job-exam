# Hoisting（提升）解答

## 重點整理

- **函式宣告**（function declaration）整個被提升，可以在宣告前呼叫。
- **`var` 宣告**被提升，但初始化留在原地，提升後值為 `undefined`。
- **`let` / `const` / `class`** 也會被提升，但處於 TDZ（暫時死區），在宣告前存取會丟 `ReferenceError`。
- **函式運算式**（如 `const fn = () => {}`）只提升變數，不提升函式體；提升行為等同其宣告關鍵字（`var` / `let` / `const`）。

## 程式碼示範

```js
foo();              // OK，印出 "foo"
function foo() { console.log('foo'); }

console.log(x);     // undefined（var 被提升）
var x = 1;

console.log(y);     // ReferenceError（TDZ）
let y = 2;
```

`foo` 因為是函式宣告，整個函式在執行前已被提升，所以可以提前呼叫；`x` 的宣告被提升但賦值留在原地，故先讀到 `undefined`；`y` 雖被提升但在宣告前位於 TDZ，存取會直接拋出 `ReferenceError`。
