# Q56 - Code Review 與重構（找碴 + 重構）

## 問題

好的前端工程師不只是「會寫」，更要能**主動發現問題、維持高品質程式碼**。這正是 Code Review 的核心能力，也是面試官很愛用的一種考法：丟一段「能動、但寫得很糟」的程式碼，看你能挑出多少問題、能不能提出有理有據的重構。

本題給你 4 段有問題的 React / TypeScript / JS 程式碼，每段都是一個獨立情境。請你扮演 **Reviewer**：

1. **找出所有問題**——不只是會壞的 bug，也包含 code smell、效能、可讀性、可及性（a11y）、型別安全、React 反模式。
2. **分類與說明**——把每個問題歸類（正確性 / 效能 / 可維護性 / a11y / 型別…），並說清楚「為什麼是問題」。
3. **重構**——寫出修正後的程式碼。

> 提示：常見的 React 反模式包含「用 `useEffect` 計算衍生狀態」「用 index 當 key / 缺 key」「直接 mutate state」「過期閉包（stale closure）」「副作用未清理」「render 內建立新物件/函式造成不必要重繪」等，看到相關結構要特別警覺。

## 情境 1. React 商品清單與購物車

這個元件會壞、也慢、也難維護。請找出問題並重構。

```tsx
import { useState, useEffect } from 'react';

type Product = { id: number; name: string; price: number };

function ProductList({ products }: { products: Product[] }) {
  const [keyword, setKeyword] = useState('');
  const [filtered, setFiltered] = useState<Product[]>([]);
  const [cart, setCart] = useState<Product[]>([]);

  // 依關鍵字過濾
  useEffect(() => {
    setFiltered(products.filter((p) => p.name.includes(keyword)));
  }, [keyword, products]);

  const addToCart = (product: Product) => {
    cart.push(product);
    setCart(cart);
  };

  return (
    <div>
      <input value={keyword} onChange={(e) => setKeyword(e.target.value)} />

      <ul>
        {filtered.map((p, i) => (
          <li key={i} style={{ padding: 8, borderBottom: '1px solid #ccc' }}>
            {p.name} - ${p.price}
            <button onClick={() => addToCart(p)}>加入購物車</button>
          </li>
        ))}
      </ul>

      <p>購物車共 {cart.length} 件</p>
    </div>
  );
}
```

**請找出問題並重構。**

## 情境 3. TypeScript 訂單資料處理

這段函式要從 API 回傳資料算出「已付款訂單的總金額」，但型別鬆散、又難讀。請找出問題並重構。

```ts
function getPaidTotal(data: any) {
  let total = 0;
  if (data) {
    if (data.orders) {
      for (let i = 0; i < data.orders.length; i++) {
        if (data.orders[i].status == 'paid') {
          if (data.orders[i].items) {
            for (let j = 0; j < data.orders[i].items.length; j++) {
              total =
                total +
                data.orders[i].items[j].price * data.orders[i].items[j].qty;
            }
          }
        }
      }
    }
  }
  return total;
}
```

**請找出問題並重構**（含：定義合適的型別、消除巢狀、命名、正確性）。

## 情境 4. React 登入表單

這個表單「看起來能用」，但對鍵盤與螢幕報讀器使用者很不友善，程式碼也塞在一起。請找出問題並重構。

```tsx
import { useState } from 'react';

function LoginForm({ onSubmit }: { onSubmit: (email: string, pwd: string) => void }) {
  const [email, setEmail] = useState('');
  const [pwd, setPwd] = useState('');
  const [error, setError] = useState('');

  const submit = () => {
    if (email === '') {
      setError('請輸入 Email');
      return;
    }
    if (pwd.length < 6) {
      setError('密碼至少 6 碼');
      return;
    }
    setError('');
    onSubmit(email, pwd);
  };

  return (
    <div>
      <div style={{ fontSize: 20 }}>登入</div>

      <div>Email</div>
      <input value={email} onChange={(e) => setEmail(e.target.value)} />

      <div>密碼</div>
      <input value={pwd} onChange={(e) => setPwd(e.target.value)} />

      {error !== '' && <div style={{ color: 'red' }}>{error}</div>}

      <div style={{ cursor: 'pointer', background: '#06f', color: '#fff' }} onClick={submit}>
        送出
      </div>
    </div>
  );
}
```

**請找出問題並重構。**
