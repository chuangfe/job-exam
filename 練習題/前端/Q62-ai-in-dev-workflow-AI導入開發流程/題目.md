# 將 AI 導入開發流程（含使用邊界）

AI 編碼工具（Copilot、Claude Code、Cursor…）已經是日常開發的一部分，愈來愈多職缺會問你：「你怎麼在工作中使用 AI？」——這其實是在測兩件事：（1）你能不能**用它變快、變好**；（2）你懂不懂**它的邊界**——什麼能給、什麼不能給、產出要怎麼把關。尤其在重視**安全、隱私、授權、保密**的團隊，「會用」還不夠，「知道界線在哪」才是加分關鍵。

本題偏觀念與實務判斷。建議先自己用白話講一遍，再對照解析。回答時請務實中立——AI 不是萬靈丹，也不是洪水猛獸，重點是「在正確的地方、用正確的方式、有把關地用」。

> 提醒：本題與 **Q56 Code Review 與重構** 互補——AI 產出的程式碼，最終仍要用 Code Review 的眼光審過。

## Q1. AI 在開發流程有哪些應用場景？

- 列出 AI 在日常開發中可以幫上忙的環節，並簡述各自怎麼用：
  - 程式碼生成（boilerplate、樣板、把想法變成初版）
  - 測試生成（單元測試、邊界案例、測試資料）
  - PR 審查輔助（自動摘要 diff、標出可疑處）
  - 文件生成（JSDoc、README、API 說明）
  - 規範／風格檢查（命名、一致性、lint 規則說明）
  - 重構（抽函式、改型別、換寫法）
  - debug 協助（讀錯誤訊息、找可疑段落、提假設）
  - commit / PR 說明（依 diff 產生訊息）
- 哪些場景「AI 特別擅長」、哪些「要格外小心」？為什麼？

## Q2. 工具生態與 CI 整合？

- 這幾類工具各自的定位是什麼、適合什麼情境：
  - GitHub Copilot（IDE 內即時補全）
  - Claude Code / Cursor（agent 型、跨檔案、能跑指令與改多個檔）
  - Code review bots（PR 上自動留言的審查機器人）
- AI 要怎麼跟 **CI/CD** 整合？（提示：把 AI 審查當成一個「非阻擋性」的參考意見，真正的 gate 還是 lint / test / type-check / 人工 review）
- 「即時補全」「對話式 agent」「PR 自動審查」三種互動模式，分別適合什麼工作？

## Q3. 使用邊界——安全、隱私、授權、保密（重點）？

- **機密資料不外流**：為什麼不能把公司原始碼、客戶資料、金鑰／密碼／token、內部架構文件直接貼給「外部雲端 AI」？可能造成什麼後果？
- **授權與著作權**：AI 產出的程式碼「來源」有什麼疑慮？為什麼大段照抄疑似有授權的產出（例如 GPL 片段）是風險？產出的著作權歸屬為什麼是灰色地帶？
- **公司政策與資料留存**：使用前該確認哪些事？（提示：這家 AI 服務會不會拿你的輸入去訓練？資料留存多久？有沒有企業版／不留存條款？公司有沒有核可清單？）
- **地端 vs 雲端模型**：什麼情況下該選地端（self-hosted / on-prem）模型？它換來什麼、犧牲什麼？
- **PII 遮罩**：非用不可、又牽涉個資時，可以怎麼降低風險？（提示：去識別化、用假資料、只貼必要片段）

## Q4. 品質把關——為什麼 AI 產出一定要人審查？

- AI 產出常見的坑有哪些？請解釋：
  - 幻覺（hallucination）——捏造不存在的 API、參數、套件
  - 過時——用了已棄用的寫法或舊版 API
  - 細微 bug——看起來對、跑起來也像對，但邊界條件錯
  - 似是而非的安全問題——例如少了輸入驗證、SQL 拼接、把祕密寫死
- 為什麼「能編譯／測試綠燈」不等於「正確」？
- 你會用哪些手段驗證 AI 產出？（提示：讀懂每一行、跑測試、加自己的邊界測試、靜態分析、小步提交）
- 「可解釋性」為什麼重要？如果你看不懂 AI 給的程式碼，該怎麼辦？

## Q5. 怎樣算「有效使用」AI？

- 什麼是好的提示（prompt）？請對照「壞提示」說明差別。（提示：明確目標、給限制、給範例、給預期輸出格式）
- 為什麼「提供上下文」是關鍵？（相關檔案、既有慣例、型別定義、錯誤訊息全文）
- 「小步驗證」是什麼意思？為什麼不要一次讓 AI 產一大坨再一次收？
- 「把 AI 當結對夥伴（pair programmer）而非取代自己判斷」具體是什麼態度？哪些決定不該外包給 AI？

## Q6. 團隊怎麼導入 AI？

- 團隊層級要制定哪些**規範**？（提示：核可工具清單、哪些資料絕對不能貼、產出審查流程、授權注意事項）
- 教育訓練該教什麼？（不只是「怎麼用」，還有「什麼不能做」）
- 怎麼**量測** AI 對生產力的影響？只看「產出行數」有什麼陷阱？
- 有哪些**風險控管**手段？（提示：企業版 + 不留存、DLP、稽核、地端模型、程式碼掃描）
