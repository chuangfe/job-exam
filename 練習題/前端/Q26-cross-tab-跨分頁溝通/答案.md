# Q26 - 跨頁 / 跨分頁溝通解答

## 方法總覽

| 方法 | 適用情境 | 同源限制 | 即時性 |
| --- | --- | --- | --- |
| BroadcastChannel | 同源多分頁廣播 | 同源 | 即時 |
| localStorage + `storage` 事件 | 同源跨分頁監聽變更 | 同源 | 即時 |
| postMessage | 跨來源 iframe / 視窗 | 可跨源 | 即時 |
| SharedWorker | 同源多分頁共享一個背景執行緒/狀態 | 同源 | 即時 |
| Service Worker | 攔截請求、推播、跨分頁中介 | 同源 | 即時 |

## 1. BroadcastChannel（最簡潔，推薦）

同源多分頁的即時廣播，API 直觀。

```tsx
// 分頁 A：發送
const channel = new BroadcastChannel("app");
channel.postMessage({ type: "logout" });

// 分頁 B：接收（發送的分頁自己不會收到）
const channel = new BroadcastChannel("app");
channel.onmessage = (e) => {
  if (e.data.type === "logout") {
    // 同步登出
  }
};
// 用完關閉
// channel.close();
```

## 2. localStorage + `storage` 事件

寫入 `localStorage` 會在**其他**同源分頁觸發 `storage` 事件（**寫入的當前分頁不會觸發**）。相容性最好。

```tsx
// 分頁 A：寫入觸發其他分頁
localStorage.setItem("logout", String(Date.now()));

// 分頁 B：監聽
window.addEventListener("storage", (e) => {
  if (e.key === "logout") {
    // 同步登出
  }
});
```

## 3. postMessage（跨來源）

用於跨來源的 iframe、`window.open` 開的視窗。**務必驗證 `origin`**，否則有安全風險。

```tsx
// 父視窗 → iframe
const iframe = document.querySelector("iframe")!;
iframe.contentWindow?.postMessage({ token: "xxx" }, "https://child.example.com");

// iframe 內接收
window.addEventListener("message", (e) => {
  // ✅ 一定要驗證來源
  if (e.origin !== "https://parent.example.com") return;
  console.log(e.data);
});
```

## 4. SharedWorker

多個同源分頁連到**同一個** worker 實例，共享狀態（例如共用一條 WebSocket 連線）。

```tsx
// 各分頁連到同一個 SharedWorker
const worker = new SharedWorker("/shared-worker.js");
worker.port.start();
worker.port.postMessage({ type: "ping" });
worker.port.onmessage = (e) => console.log(e.data);
```

## 5. Service Worker

可攔截網路請求、做離線快取、推播通知，也能作為多分頁的中介，透過 `clients` 廣播訊息給所有受控分頁。

```tsx
// Service Worker 內：向所有受控分頁廣播
self.addEventListener("push", async () => {
  const all = await self.clients.matchAll();
  all.forEach((client) => client.postMessage({ type: "new-data" }));
});
```

## 選用建議

- 同源、只是要互相通知 → **BroadcastChannel**（首選），需相容舊瀏覽器則退回 **localStorage + storage 事件**。
- 跨來源（iframe / 彈窗） → **postMessage**，且必須驗證 `origin`。
- 要共享連線/運算或離線能力 → **SharedWorker** / **Service Worker**。
