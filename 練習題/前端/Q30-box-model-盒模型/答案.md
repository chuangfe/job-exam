# CSS 盒模型（Box Model）解答

## 盒模型組成
每個元素都可視為一個矩形盒子，由內而外包含四層：
- **content（內容）**：實際內容區域（文字、圖片等）。
- **padding（內距）**：內容與邊框之間的空間。
- **border（邊框）**：包住 padding 的邊框。
- **margin（外距）**：盒子與其他元素之間的空間。

## content-box 與 border-box 差異
- **content-box（預設值）**：`width` 只計算「內容」的寬度，`padding` 與 `border` 會「另外加上去」。
- **border-box**：`width` 已經包含 `content + padding + border`，比較直覺；內容區會自動縮小以容納 padding/border。

### 範例對照
```css
.box {
  width: 200px;
  padding: 20px;
  border: 5px solid #333;
}

/* content-box（預設）：實際佔據寬度 = 200 + 20*2 + 5*2 = 250px */
.box-content {
  box-sizing: content-box;
}

/* border-box：實際佔據寬度 = 200px（內容區自動縮成 150px） */
.box-border {
  box-sizing: border-box;
}
```

## 為什麼常用 `* { box-sizing: border-box; }`
- 設定的 `width` 就是元素最終佔據的寬度，不必再手動加減 padding/border，計算更直覺。
- 做百分比寬度（例如兩欄各 `width: 50%` 再加 padding）時不會因為加了 padding 而撐爆換行。
- 全站統一行為，避免不同元件混用兩種模式造成的對齊問題。

```css
/* 常見全域設定，連偽元素一起涵蓋 */
*,
*::before,
*::after {
  box-sizing: border-box;
}
```
