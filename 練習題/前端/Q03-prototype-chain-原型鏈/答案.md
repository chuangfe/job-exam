# 原型與原型鏈（prototype chain）解答

## 重點整理

- 每個函式都有 `prototype` 屬性；每個物件都有內部 `[[Prototype]]`（可由 `__proto__` 或 `Object.getPrototypeOf` 存取）。
- 存取屬性時會沿著原型鏈往上找，找不到回傳 `undefined`；到 `Object.prototype` 的 `[[Prototype]]` 為 `null` 時結束查找。
- `new` 做的事：
  1. 建立空物件。
  2. 物件的 `[[Prototype]]` 指向建構式的 `prototype`。
  3. 以新物件為 `this` 執行建構式。
  4. 若建構式回傳非物件，則回傳該新物件（否則回傳建構式回傳的物件）。
- `class` 是原型繼承的語法糖；`extends` 設定原型鏈，`super` 呼叫父建構式。
- `instanceof` 檢查建構式的 `prototype` 是否在物件的原型鏈上。

## 程式碼示範

```js
function Animal(name) { this.name = name; }
Animal.prototype.speak = function () { return `${this.name} makes a sound`; };
const a = new Animal('cat');
a.speak();                              // "cat makes a sound"
Object.getPrototypeOf(a) === Animal.prototype; // true
a instanceof Animal;                    // true
```

呼叫 `a.speak()` 時，`a` 本身沒有 `speak`，於是沿原型鏈往上在 `Animal.prototype` 找到並執行，此時 `this` 為 `a`。
