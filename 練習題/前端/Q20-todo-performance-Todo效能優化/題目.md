# Todo 效能優化

## 問題
有一個 Todo 清單，每個項目可點擊切換完成狀態。目前的實作中，只要切換「其中一項」，整個清單的所有項目都會重新渲染（可在子元件加 `console.log` 觀察）。

請優化，使得切換某一項時，只有「那一項」重新渲染。

## 提示
要同時做到以下三件事，缺一不可：
1. 把單一項目拆成獨立子元件，並用 `React.memo` 包起來（以 props 淺比較跳過重渲染）。
2. 傳給子元件的回呼函式參考要「穩定」——否則每次父層重渲染都產生新函式，memo 會失效。
   - 思考：如果 `onToggle` 依賴 `todos`，依賴陣列就會一直變。要怎麼讓依賴陣列為空又能更新到最新資料？（提示：函式式更新 `setTodos(prev => ...)`）
3. 列表要使用「穩定且唯一」的 key（用 id，不要用 index）。

請想想：如果只做了 memo 但沒穩定回呼，會發生什麼？如果回呼穩定了但 key 用 index 又會如何？
