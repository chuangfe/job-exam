# Todo 效能優化 解析

## 完整程式碼

```tsx
import { memo, useCallback, useState } from 'react';

interface Todo { id: number; text: string; done: boolean; }

const TodoItem = memo(function TodoItem({
  todo, onToggle,
}: { todo: Todo; onToggle: (id: number) => void }) {
  console.log('render', todo.id);
  return (
    <li onClick={() => onToggle(todo.id)}
        style={{ textDecoration: todo.done ? 'line-through' : 'none' }}>
      {todo.text}
    </li>
  );
});

function TodoList() {
  const [todos, setTodos] = useState<Todo[]>([]);

  // 用函式式更新，依賴陣列為空 → onToggle 參考永遠穩定 → memo 子項不會無謂重渲染
  const onToggle = useCallback((id: number) => {
    setTodos((prev) =>
      prev.map((t) => (t.id === id ? { ...t, done: !t.done } : t))
    );
  }, []);

  return (
    <ul>
      {todos.map((t) => (
        <TodoItem key={t.id} todo={t} onToggle={onToggle} />
      ))}
    </ul>
  );
}
```

## 重點解析

三件事缺一不可：

1. **memo 子元件**：`TodoItem` 用 `React.memo` 包裝，props 沒變就跳過渲染。
2. **useCallback 穩定回呼**：`onToggle` 用 `useCallback` 且依賴陣列為空。關鍵在於改用「函式式更新」`setTodos(prev => ...)`，因此不需要把 `todos` 放進依賴，回呼參考永遠穩定，memo 才不會因為每次拿到新函式而失效。
3. **穩定的 key**：列表用 `t.id` 當 key，讓 React 正確對應每個項目的 Fiber。

否則切換一項會重渲染整列：
- 只 memo、回呼不穩定 → 每次父層渲染都產生新 `onToggle`，淺比較判定 props 變了，所有項目照樣重渲染。
- 回呼穩定但 key 用 index → 插入/刪除/排序時 index 與內容對不上，造成狀態錯位與非必要重渲染。
