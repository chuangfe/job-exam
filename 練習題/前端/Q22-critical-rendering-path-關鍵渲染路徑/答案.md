# Q22 - Critical Rendering Path（關鍵渲染路徑）解答

## 一句話總結

關鍵渲染路徑是瀏覽器把 HTML、CSS、JS 轉換成螢幕上實際像素的一連串步驟。優化的核心是「縮短、縮小、延後」關鍵資源，讓首次畫面（first paint）盡快出現。

## 完整流程

```
HTML  ──解析──▶  DOM Tree
                          ╲
                           ╲（合併）
                            ▶  Render Tree  ──▶  Layout  ──▶  Paint  ──▶  Composite
                           ╱     （只含可見節點）   （計算位置/大小）  （填顏色/像素）  （合成圖層上螢幕）
CSS   ──解析──▶  CSSOM    ╱
```

1. **DOM（Document Object Model）**：解析 HTML，建立節點樹。
2. **CSSOM（CSS Object Model）**：解析 CSS，建立樣式樹。
3. **Render Tree（渲染樹）**：合併 DOM 與 CSSOM，只包含「會顯示」的節點（`display: none` 不會進渲染樹，但 `visibility: hidden` 會）。
4. **Layout（佈局 / reflow）**：計算每個節點在視窗中的精確位置與大小（幾何資訊）。
5. **Paint（繪製）**：把節點填上實際像素（顏色、文字、邊框、陰影等）。
6. **Composite（合成）**：將多個圖層（layers）依正確順序合成，輸出到螢幕。

## 阻塞行為

- **CSS 阻塞渲染（render-blocking）**：瀏覽器必須等 CSSOM 建立完成才能產生 Render Tree，否則畫面可能出現「無樣式內容閃爍（FOUC）」。因此 CSS 預設會阻塞渲染。
- **JS 阻塞解析（parser-blocking）**：遇到沒有 `async` / `defer` 的 `<script>` 時，HTML 解析會暫停，先下載並執行 JS（因為 JS 可能操作 DOM/CSSOM），執行完才繼續解析。
- 此外，若 JS 之前有尚未載入完的 CSS，JS 還會等待 CSS 載入完才執行（因為 JS 可能讀取樣式）。

## 優化方式

### 1. CSS 放 `<head>`、盡早載入

```css
/* 關鍵 CSS（首屏需要的樣式）可內聯（inline）到 <head>，避免額外請求 */
```

```tsx
{/* 非關鍵 CSS 用 media 技巧或 preload 延後，不阻塞首屏 */}
<link
  rel="preload"
  href="/non-critical.css"
  as="style"
  onLoad={(e) => { (e.currentTarget as HTMLLinkElement).rel = "stylesheet"; }}
/>
```

### 2. JS 用 `defer` / `async` 或放底部

```tsx
{/* defer：照順序、在 DOM 解析完成後、DOMContentLoaded 前執行（推薦用於有相依順序的腳本） */}
<script defer src="/app.js" />

{/* async：下載完立即執行、不保證順序（適合獨立的第三方腳本，如分析工具） */}
<script async src="/analytics.js" />
```

| 屬性 | 下載 | 執行時機 | 保證順序 |
| --- | --- | --- | --- |
| 無 | 阻塞解析 | 立即 | 是 |
| `async` | 不阻塞 | 下載完立即（可能打斷解析） | 否 |
| `defer` | 不阻塞 | 解析完成後依序 | 是 |

### 3. 減少關鍵資源的數量與大小

- 移除未使用的 CSS/JS（tree-shaking、code splitting）。
- 壓縮（minify）+ Gzip/Brotli 壓縮傳輸。
- 用 CDN 縮短傳輸距離。
- 對首屏關鍵資源用 `<link rel="preload">` 提前載入。

## 面試加分

- 「first paint」「first contentful paint」之所以慢，常常就是被 render-blocking CSS 或 parser-blocking JS 卡住。
- 現代框架的 SSR / 串流（streaming）就是為了讓使用者更早看到內容，縮短關鍵渲染路徑的感知時間。
