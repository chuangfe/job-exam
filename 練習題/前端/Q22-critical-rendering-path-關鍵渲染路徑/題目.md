# Q22 - Critical Rendering Path（關鍵渲染路徑）

## 問題

Critical Rendering Path（關鍵渲染路徑）是什麼？瀏覽器從拿到 HTML 到把畫面顯示在螢幕上，中間經過哪些步驟？以及如何優化關鍵渲染路徑？

## 引導思考

- 瀏覽器拿到 HTML 後會建立什麼樣的資料結構？CSS 又會建立什麼？這兩者如何合併？
- 從合併後的結構到實際出現在螢幕上，還要經過哪幾個階段？每個階段做什麼？
- 哪些資源會「阻塞渲染（render-blocking）」？哪些會「阻塞解析（parser-blocking）」？
- `<script>` 沒有加 `async` / `defer` 時會發生什麼事？
- CSS 應該放在哪裡？JS 應該放在哪裡？為什麼？
- 要減少首次畫面出現的時間，可以從「關鍵資源的數量、大小、傳輸距離」三個面向各做哪些事？
