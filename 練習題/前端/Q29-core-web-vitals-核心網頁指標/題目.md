# Q29 - Core Web Vitals（核心網頁指標）

## 問題

Core Web Vitals（核心網頁指標）中的 LCP、CLS、INP 各代表什麼？各自衡量哪個面向（載入、視覺穩定、互動）？目標值是多少？以及各自有哪些優化方式？

## 引導思考

- LCP（Largest Contentful Paint）衡量的是什麼？「最大內容」通常是頁面上的哪個元素？好的門檻是幾秒？
- LCP 太慢通常是被什麼拖累（主圖太大、伺服器太慢、阻塞資源）？可以怎麼改善？
- CLS（Cumulative Layout Shift）衡量什麼？什麼情況會造成版面突然位移？目標值多少？
- 圖片、字體、動態插入的內容，分別會怎麼造成 CLS？要如何避免？
- INP（Interaction to Next Paint）是 2024 年取代誰的指標？它衡量什麼？目標值多少？
- INP 太差通常代表主執行緒被什麼卡住？React 有哪些 API（如 `useDeferredValue`、`startTransition`）或手段（拆分長任務、Web Worker）可以改善？
