# Q29 - Core Web Vitals（核心網頁指標）解答

## 三大指標總覽

| 指標 | 全名 | 衡量面向 | 目標值（良好） |
| --- | --- | --- | --- |
| LCP | Largest Contentful Paint | 載入速度 | < 2.5s |
| CLS | Cumulative Layout Shift | 視覺穩定性 | < 0.1 |
| INP | Interaction to Next Paint | 互動回應性 | < 200ms |

## LCP（最大內容繪製）— 載入

頁面主視覺（通常是最大的圖片、影片或大段文字區塊）繪製完成的時間，代表使用者「感覺載入完成」的時點。

**優化方式：**

- 壓縮主圖、用現代格式（WebP / AVIF），並 `preload` 關鍵主圖。
- 用 CDN 縮短傳輸距離、降低伺服器回應時間（TTFB）。
- SSR / 預渲染讓內容更早出現。
- 移除阻塞渲染的 CSS / JS（見 Q22）。
- 首屏 LCP 圖片**不要懶載入**（見 Q28）。

```tsx
{/* preload 首屏最大主圖，搶先載入 */}
<link rel="preload" as="image" href="/hero.avif" fetchPriority="high" />
```

## CLS（累積版面位移）— 視覺穩定

頁面在載入過程中元素「無預期跳動」的累積程度。值越低越穩定。

**常見成因與對策：**

- **圖片/影片沒設尺寸** → 設 `width`/`height` 或 `aspect-ratio` 預留空間。
- **字體載入造成文字重排（FOUT/FOIT）** → 用 `font-display: optional` 或 `swap`。
- **動態插入內容（廣告、橫幅）把既有內容推開** → 預留固定空間，避免插在已可見內容上方。

```css
/* 圖片預留空間，避免載入後推擠 */
img {
  aspect-ratio: 16 / 9;
  width: 100%;
  height: auto;
}

/* 字體載入策略，減少重排 */
@font-face {
  font-family: "MyFont";
  src: url("/my-font.woff2") format("woff2");
  font-display: optional; /* 或 swap */
}
```

## INP（互動到下次繪製）— 互動性

衡量使用者互動（點擊、輸入等）到畫面實際更新之間的延遲。**2024 年正式取代 FID（First Input Delay）**，比 FID 更全面（涵蓋整個頁面生命週期的所有互動，而非只看第一次）。

**優化方式：核心是讓主執行緒不被長任務（long task）卡住。**

- 拆分長任務（把大運算切成多個小區塊，讓出主執行緒）。
- 用 React 並行特性把非急迫更新標為低優先：`startTransition` / `useDeferredValue`。
- 減少主執行緒上的 JS（程式碼分割、移除不必要的運算）。
- 把重運算移到 **Web Worker**，不阻塞主執行緒。

```tsx
import { useState, useDeferredValue, startTransition } from "react";

function Search({ allItems }: { allItems: string[] }) {
  const [query, setQuery] = useState("");
  // 輸入框立即更新（急迫），昂貴的篩選結果延後（不阻塞輸入回應）
  const deferredQuery = useDeferredValue(query);
  const results = allItems.filter((i) => i.includes(deferredQuery));

  return (
    <>
      <input
        value={query}
        onChange={(e) => {
          const v = e.target.value;
          setQuery(v); // 高優先：保持輸入流暢
          startTransition(() => {
            // 低優先：可被中斷的非急迫更新
          });
        }}
      />
      <ul>{results.map((r) => <li key={r}>{r}</li>)}</ul>
    </>
  );
}
```

## 重點回顧

- LCP 看**載入**、CLS 看**穩定**、INP 看**互動**——三者分別對應「快、穩、順」。
- INP 自 2024 起取代 FID，是現在衡量互動體驗的主指標。
