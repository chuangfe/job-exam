# Closure（閉包）解答

## 什麼是閉包

閉包 = 函式 + 函式被建立時所處的詞法環境（lexical environment）。內層函式即使在外層函式回傳後，仍能存取外層作用域的變數。

## 本質

變數因為被內層函式參考，無法被 GC（垃圾回收）回收，因此閉包能夠「記住」狀態。換句話說，閉包讓函式可以攜帶一份對外層變數的參考，跨越外層函式的生命週期繼續存取。

## 常見用途

- 模組化封裝私有變數（用閉包隱藏內部狀態，只暴露需要的介面）。
- 函式工廠（用相同邏輯產生帶有不同設定的函式）。
- debounce / throttle 等需要保存計時器或上次執行時間的場景。
- React Hooks（`useState` 內部就是靠閉包保存值）。
- 柯里化（Currying）。

## 經典陷阱

迴圈中用 `var` 註冊回呼，所有回呼會共用同一個變數（函式作用域只有一個 `i`）。

```js
// 陷阱：印出 3, 3, 3
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// 解法 1：用 let（每次迭代產生新的綁定）→ 0, 1, 2
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// 解法 2：IIFE 建立新作用域
for (var i = 0; i < 3; i++) {
  ((j) => setTimeout(() => console.log(j), 0))(i);
}
```

第一段印出 `3, 3, 3`，是因為 `setTimeout` 的回呼在迴圈結束後才執行，而此時三個回呼共用的 `i` 已經是 `3`。`let` 會在每次迭代建立新的區塊綁定，IIFE 則用立即執行函式為每次迭代建立獨立的作用域，因此都能得到 `0, 1, 2`。
