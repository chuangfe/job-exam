# Q24 - HTTP 快取（Cache-Control / ETag）

## 問題

HTTP 快取是如何運作的？「強快取（strong cache）」與「協商快取（negotiated cache）」有什麼差別？實務上對 HTML 和帶 hash 的靜態資源應該分別用什麼快取策略？

## 引導思考

- 瀏覽器發出請求前，會先檢查什麼？什麼情況下「根本不發請求」就直接用本地副本？
- `Cache-Control: max-age` 和 `Expires` 各代表什麼？兩者誰優先？
- 當強快取過期後，瀏覽器會做什麼？「協商」是在跟誰協商、協商什麼？
- `ETag` / `If-None-Match` 與 `Last-Modified` / `If-Modified-Since` 各是怎麼配對運作的？
- 伺服器回 `304 Not Modified` 時，有沒有傳回 body？有什麼好處？
- 為什麼 HTML 通常用 `no-cache`，而檔名帶 hash 的 JS/CSS 可以用 `max-age=31536000, immutable`？
