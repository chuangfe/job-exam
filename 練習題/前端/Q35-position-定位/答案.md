# CSS position（定位）解答

## 各值差異
- **static（預設）**：正常文件流排列，`top/right/bottom/left` 無效。
- **relative**：相對「自身原本位置」做偏移；**仍佔據原本空間**（原位保留，視覺上位移）。常用來當作 absolute 子元素的定位基準。
- **absolute**：脫離文件流（不佔空間），相對「最近的非 static 祖先」定位；若找不到，則相對初始包含區塊（viewport / 根元素）。
- **fixed**：脫離文件流，相對「視窗（viewport）」定位，捲動時固定不動（常用於回到頂部按鈕、固定導覽列）。
- **sticky**：混合行為——在捲動到設定的閾值之前表現為 `relative`，達到閾值後「黏住」表現為 `fixed`，超出父容器範圍後又恢復。

## 程式碼示範
```css
/* relative：偏移但仍佔位，並作為子元素的定位基準 */
.parent {
  position: relative;
}

/* absolute：相對最近的非 static 祖先（此處為 .parent）定位 */
.parent .badge {
  position: absolute;
  top: 0;
  right: 0;
}

/* fixed：相對視窗固定 */
.back-to-top {
  position: fixed;
  bottom: 24px;
  right: 24px;
}

/* sticky：捲動到距頂端 0 時黏住 */
.toc {
  position: sticky;
  top: 0;
}
```

## 重點補充
- 要讓 `absolute` 子元素相對某個父層定位，**該父層必須設定非 static 的 position**（通常用 `position: relative;`），否則會往上找其他祖先或回到視窗。
- `fixed` 永遠相對視窗；`sticky` 受限於其父容器範圍，超出父容器就不再黏住，這是兩者最大的差別。
