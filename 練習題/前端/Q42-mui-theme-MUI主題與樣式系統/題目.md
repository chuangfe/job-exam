# MUI（v5+）主題與樣式系統

資深「設計系統前端」面試常考。MUI（Material UI v5/v6）底層改用 Emotion，主題（theme）成為整套設計系統的單一真相來源（single source of truth）。請依序回答；觀念題先用白話講出來，實作題請「實際寫出程式碼」（可貼到 CodeSandbox / MUI Playground 驗證）。建議先自己作答，再對照解析。

## Q1. `createTheme` 與主題結構
- MUI 的 theme 物件包含哪些主要區塊？請說明 `palette`、`typography`、`spacing`、`breakpoints`、`shape`、`shadows`（elevation）、`zIndex` 各自負責什麼。
- `spacing` 預設值是多少？`theme.spacing(2)` 會算出什麼？
- `breakpoints` 預設有哪些斷點？`theme.breakpoints.up('md')` 會產生什麼字串？
- `shadows` 陣列與元件的 `elevation` prop 有什麼關係（例如 `<Paper elevation={3} />`）？
- 請寫一個 `createTheme`，自訂主色/次色、字體家族、`shape.borderRadius`，並新增一個自訂 palette 色 `brand`。

## Q4. `styled()`（Emotion）建立樣式化元件
