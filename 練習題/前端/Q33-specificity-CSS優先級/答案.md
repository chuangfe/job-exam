# CSS Specificity（優先級）解答

## 優先順序（由高到低）
1. `!important`
2. 行內樣式（inline style，例如 `style="..."`）
3. id 選擇器（`#id`）
4. class / 屬性 / 偽類選擇器（`.class`、`[attr]`、`:hover`）
5. 元素 / 偽元素選擇器（`div`、`::before`）

## 計算方式
Specificity 可視為三組計數 **(id, class, element)**，由左到右逐項比較大小：
- 計算每條選擇器中 id 的數量、class/屬性/偽類的數量、元素/偽元素的數量。
- 先比第一組（id），相同再比第二組（class），再比第三組（element）。
- **若完全相同分數，則「後出現」的規則覆蓋「先出現」的（後者優先）。**

### 範例
```css
/* (1, 1, 1) → id=1, class=1, element=1 */
#nav .item span { color: red; }

/* (0, 3, 0) → class/偽類共 3 個 */
.item .text.active { color: blue; }
```
比較第一組 id：`1 > 0`，因此 `#nav .item span` 勝出，文字為紅色（即使另一條 class 較多）。

## 為什麼避免濫用 `!important`
- 它會凌駕一般計算規則，後續要再覆蓋時往往只能再疊一個 `!important`，造成惡性循環、難以維護。
- 讓樣式來源變得難以追蹤、debug 困難。
- 較好的做法：以合理的選擇器結構與適度的 specificity 管理樣式，把 `!important` 留給極少數真正必要的覆蓋情境。
