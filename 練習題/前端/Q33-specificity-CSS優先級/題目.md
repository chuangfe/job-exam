# CSS Specificity（優先級）

## 問題
1. 當多條 CSS 規則套用到同一個元素時，瀏覽器如何決定哪一條勝出？
2. CSS specificity（優先級）是如何「計算」的？
3. `!important`、行內樣式、id、class、元素選擇器之間的優先順序為何？
4. 為什麼應該盡量避免濫用 `!important`？

## 引導思考
- 若 specificity 相同，最後勝出的會是哪一條規則？
- `#id .item span` 與 `.item .text.active` 哪一個優先級較高？該怎麼比？
- 如果你發現必須用 `!important` 才能覆蓋樣式，這通常代表什麼問題？
