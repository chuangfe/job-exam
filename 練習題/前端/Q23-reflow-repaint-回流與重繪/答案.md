# Q23 - reflow（回流） vs repaint（重繪）解答

## 核心差異

| | reflow（回流 / layout） | repaint（重繪） |
| --- | --- | --- |
| 觸發原因 | 幾何屬性變更（寬高、位置、字體大小、視窗 resize、新增/刪除節點） | 外觀變更但不影響佈局（顏色、背景、`visibility`、`box-shadow`） |
| 做的事 | 重新計算節點的位置與大小 | 重新填上像素 |
| 成本 | 昂貴（可能連帶影響父/子/兄弟節點） | 較便宜 |
| 連帶關係 | reflow 後**一定**會 repaint | repaint **不一定**會 reflow |

簡記：**改「形狀/位置」→ reflow + repaint；只改「外觀」→ repaint。**

## 為什麼 reflow 昂貴

改一個元素的幾何屬性，可能讓整棵渲染樹的佈局需要重算（例如改一個容器的寬度，會牽動所有子元素重新排版）。

## Layout Thrashing（佈局抖動 / 強制同步佈局）

在迴圈中「讀 → 寫 → 讀 → 寫」交錯操作 DOM，會逼瀏覽器每次讀取時都立刻 reflow（因為前一步的寫入讓佈局變髒），導致反覆強制同步佈局。

```tsx
// ❌ 壞：每次讀 offsetWidth 都觸發一次強制同步佈局
function bad(boxes: HTMLElement[]) {
  for (const box of boxes) {
    const w = box.offsetWidth; // 讀（強制 reflow）
    box.style.width = w + 10 + "px"; // 寫（弄髒佈局）
  }
}

// ✅ 好：先全部讀，再全部寫（批次讀寫）
function good(boxes: HTMLElement[]) {
  const widths = boxes.map((b) => b.offsetWidth); // 先一次讀完
  boxes.forEach((b, i) => {
    b.style.width = widths[i] + 10 + "px"; // 再一次寫完
  });
}
```

## 優化方式

### 1. 批次讀寫 DOM、避免強制同步佈局

如上例。需要讀取佈局資訊（`offsetTop`、`getBoundingClientRect()` 等）時集中處理。

### 2. 用 `transform` / `opacity` 走合成層

`transform` 與 `opacity` 不觸發 reflow 或 repaint，直接在合成階段由 GPU 處理，動畫最流暢。

```css
/* ❌ 改 left 會 reflow */
.move-bad {
  transition: left 0.3s;
  left: 100px;
}

/* ✅ 改 transform 走合成層，不 reflow 也不 repaint */
.move-good {
  transition: transform 0.3s;
  transform: translateX(100px);
  will-change: transform; /* 提示瀏覽器提前建立合成層（適度使用） */
}
```

### 3. 用 class 一次改多個樣式

```tsx
// ❌ 逐項改，可能多次 reflow
el.style.width = "100px";
el.style.height = "100px";
el.style.margin = "10px";

// ✅ 加一個 class，一次套用
el.classList.add("resized");
```

### 4. 用 DocumentFragment 批次插入節點

```tsx
const frag = document.createDocumentFragment();
for (let i = 0; i < 100; i++) {
  const li = document.createElement("li");
  li.textContent = `item ${i}`;
  frag.appendChild(li); // 操作 fragment 不觸發 reflow
}
list.appendChild(frag); // 只觸發一次 reflow
```

### 其他技巧

- 對要動畫/頻繁變動的元素，先 `position: absolute` 脫離正常文檔流，縮小 reflow 影響範圍。
- 隱藏元素時若要做大量 DOM 操作，可先 `display: none`（一次 reflow），改完再顯示（再一次 reflow），中間操作不觸發 reflow。
