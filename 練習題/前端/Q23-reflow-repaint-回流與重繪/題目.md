# Q23 - reflow（回流） vs repaint（重繪）

## 問題

reflow（回流）與 repaint（重繪）有什麼差異？哪一個比較昂貴？實務上要如何優化，避免效能問題？

## 引導思考

- 改變哪些 CSS 屬性會觸發 reflow？改變哪些只會觸發 repaint？
- reflow 一定會接著發生 repaint 嗎？反過來呢？
- 為什麼 reflow 比 repaint 昂貴？
- 什麼是「強制同步佈局（forced synchronous layout）」或「layout thrashing（佈局抖動）」？什麼樣的程式碼會造成它？
- 「批次讀、批次寫」DOM 為什麼能改善效能？
- 為什麼 `transform` 和 `opacity` 的動畫比改 `top` / `left` / `width` 流暢？這跟「合成層（composite layer）」有什麼關係？
