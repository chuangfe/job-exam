# Q28 - 圖片懶載入（Lazy Loading）解答

## 為什麼要懶載入

頁面上不在視窗內的圖片先不下載，等到「即將進入視窗」才載入。好處：縮短首屏載入時間、節省使用者頻寬、減少不必要的請求。

## 做法一：原生 `loading="lazy"`（最簡單）

```tsx
<img src="/photo.jpg" alt="風景" loading="lazy" width={800} height={600} />
```

- 一行搞定，瀏覽器自動處理。
- 限制：無法自訂「提前多遠開始載入」的距離門檻，行為由瀏覽器決定；對需要精細控制或佔位邏輯的場景不夠用。

## 做法二：IntersectionObserver（進階、可控）

把真正的網址放在 `data-src`，當圖片進入視窗時才把它設給 `src` 觸發載入，載入後即停止觀察。

```tsx
import { useEffect, useRef, useState } from "react";

function LazyImage({ src, alt }: { src: string; alt: string }) {
  const ref = useRef<HTMLImageElement>(null);
  const [shown, setShown] = useState(false);

  useEffect(() => {
    const io = new IntersectionObserver(([entry]) => {
      if (entry.isIntersecting) {
        setShown(true);
        io.disconnect(); // 載入後不必再觀察
      }
    });
    if (ref.current) io.observe(ref.current);
    return () => io.disconnect();
  }, []);

  return <img ref={ref} src={shown ? src : undefined} alt={alt} loading="lazy" />;
}
```

可透過 `rootMargin` 提前載入，改善體感（例如提前 200px 開始載）：

```tsx
const io = new IntersectionObserver(
  ([entry]) => {
    if (entry.isIntersecting) {
      setShown(true);
      io.disconnect();
    }
  },
  { rootMargin: "200px" } // 進入視窗前 200px 就開始載入
);
```

## 關鍵：避免 CLS（版面位移）

懶載入的圖片在載入前若沒有佔位空間，圖片一出現會把下方內容往下推，造成累積版面位移（CLS）。務必預留尺寸：

```css
/* 方法 A：用 aspect-ratio 依比例預留空間（推薦，響應式友善） */
.lazy-img {
  width: 100%;
  aspect-ratio: 4 / 3;
  object-fit: cover;
  background: #eee; /* 載入前的佔位底色 */
}
```

```tsx
{/* 方法 B：直接給 width / height 屬性，瀏覽器會據此預留空間 */}
<img src={src} alt={alt} loading="lazy" width={800} height={600} />
```

## 加分細節

- 首屏「最大內容圖（LCP 圖片）」**不要**懶載入，反而應 `loading="eager"` 或 `preload`，否則拖慢 LCP。
- 可搭配低畫質縮圖（LQIP / blur placeholder）先顯示模糊圖，提升體感。
- `decoding="async"` 讓圖片解碼不阻塞主執行緒。
