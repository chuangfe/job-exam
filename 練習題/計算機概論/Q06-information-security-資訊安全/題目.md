## 觀念速補

- **CIA 三要素**：機密性(Confidentiality 不該看的人看不到)、完整性(Integrity 資料沒被竄改)、可用性(Availability 需要時用得到)。資安一切目標都繞著這三個。
- **對稱 vs 非對稱加密**：
  - 對稱：加解密用「同一把鑰匙」，快，但鑰匙怎麼安全給對方是難題（如 AES）。
  - 非對稱：公鑰加密、私鑰解密（或反過來簽章），慢但解決金鑰交換（如 RSA）。HTTPS 用「非對稱交換出一把對稱金鑰，之後用對稱傳資料」兼顧安全與速度。
- **雜湊(Hash)**：單向、不可逆，同輸入必同輸出（如 SHA-256）。用於驗證完整性、存密碼（要加 salt）。注意：雜湊**不是加密**（不能解回去）。
- **常見攻擊（前端視角）**：
  - **XSS**：把惡意 JS 注入網頁讓別人瀏覽器執行。防：輸出編碼、CSP、別用 `dangerouslySetInnerHTML` 塞未過濾內容。
  - **CSRF**：誘使已登入使用者在不知情下送出請求。防：CSRF token、SameSite cookie。
  - **SQL Injection**：把惡意 SQL 拼進查詢。防：參數化查詢(prepared statement)、別字串拼接 SQL。
  - **DDoS**：大量流量灌爆服務（攻可用性）。防：流量清洗、CDN、限流。
