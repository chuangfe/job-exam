# 計算機概論 — 題目目錄（弱點補強軌道）

> 考試制：中華電信／國營／郵政
> 對象：會寫程式的前端工程師（React 主力，懂一點 Node），但沒受過正式 CS 訓練。
> 本軌道屬於**弱點補強軌道**：偏觀念／選擇／簡答題，**非可執行程式**，故以 markdown 呈現。
>
> 心法：你已經會的東西（HTTP、JSON、async、XSS/CSRF、瀏覽器事件迴圈）其實涵蓋了計概一大半，只是沒人幫你把名詞對起來。這份就是幫你「把工作經驗翻譯成考試術語」。

---

## 結構說明

- 每個主題一個資料夾（不是每題一夾），命名 `Q<編號>-<英文slug>-<中文短名>/`。
- 每個資料夾內兩檔：
  - `題目.md`：該主題「觀念速補」摘要 + 所有自我測驗題目（**不含答案**）。
  - `答案.md`：逐題答案 + 解析。
- 練法：先看 `題目.md` 的觀念速補補底子，再做自我測驗，做完一題再翻 `答案.md` 對解析。

## 建議練習順序

依「CP 值 + 補弱」原則安排：

1. **Q01 作業系統** — 超高頻，計算題（排程、置換）幾乎每年出，先把這塊拿穩。
2. **Q07 新興科技** — 中華電信必考，且多為名詞配對（5G 三情境、雲端三層），背熟就送分。
3. **Q04 網路** — 你工作經驗已涵蓋大半，順手鞏固並補子網路計算。
4. **Q06 資訊安全** — 從你熟的 XSS/CSRF 延伸到 CIA、對稱/非對稱加密。
5. **Q03 資料庫** — SQL 你會寫，重點補正規化（1NF~BCNF）與 ACID。
6. **Q02 資料結構觀念** — Big-O、樹走訪、運算式轉換，多動手算。
7. **Q05 計算機組織** — 最弱但考點固定（進位轉換、布林代數、記憶體階層），最後衝刺。

## 應試小提醒

- **計算題（排程、置換、進位、子網路）幾乎每年出**，務必動手算到熟，這是 CP 值最高的拿分區。
- **名詞配對**（OSI 各層協定、ACID、CIA、5G 三情境、雲端三層）背熟就送分。
- 善用你的工作經驗對應術語：HTTP/TCP、XSS/CSRF、Vercel(PaaS)、JS 單執行緒(事件迴圈)、Map(雜湊)——很多概念你早就會，只是要對上考試的講法。
