# Q04 網路 — 答案與解析

> 題目請見同資料夾的 `題目.md`。

---

**Q1（選擇）** HTTP、DNS、FTP 屬於哪一層？

**答案：C**。這些是應用層協定。TCP/UDP 在傳輸層，IP 在網路層，MAC/乙太網在資料連結層。

---

**Q2（選擇）** 何者最適合使用 UDP？

**答案：C**。即時應用重視低延遲，掉封包可容忍，用 UDP。其餘需要可靠傳輸用 TCP。DNS 查詢也常用 UDP（小、快）。

---

**Q3（簡答）** TCP 三方交握三步驟與目的。

1. Client → Server：**SYN**（我想連，序號 x）
2. Server → Client：**SYN + ACK**（我收到了，我也想連，序號 y，確認 x+1）
3. Client → Server：**ACK**（確認 y+1，連線建立）

目的：在傳資料前，雙方互相確認「對方收送都正常、同步初始序號」，建立可靠連線。連線結束則是四次揮手(FIN/ACK ×2)。

---

**Q4（手算／選擇）** `192.168.1.0/24` 可用主機數。

**答案：A**。`/24` 主機部分有 8 bit → 2⁸ = 256 個位址，但要扣掉「網路位址(.0)」與「廣播位址(.255)」 → 256 − 2 = **254** 個可用主機。

---

**Q5（手算／簡答）** `192.168.10.0/26` 切割。

- `/26` → 主機 bit = 32 − 26 = 6 → 2⁶ − 2 = **62 個可用主機**。
- 從 /24 切到 /26，多借 2 bit → 2² = **4 個子網**（.0、.64、.128、.192 各一段）。

---

**Q6（選擇）** 域名轉 IP 的協定。

**答案：B**。DNS 做域名↔IP 解析。DHCP 是自動分配 IP 給裝置；ARP 是 IP↔MAC 位址解析（同一區網內）。

---

**Q7（選擇）** 自動取得 IP/遮罩/閘道/DNS 的協定。

**答案：B**。DHCP（動態主機設定協定）。NAT 做私有/公有 IP 轉換；ICMP 是 ping 用的（回報錯誤/診斷）。

---

**Q8（簡答）** 輸入網址到看到網頁的網路步驟。

1. **DNS 解析**：把網域名稱查成 IP。
2. **TCP 三方交握**：與伺服器建立連線（HTTPS 還會做 TLS 交握）。
3. **送出 HTTP 請求**：GET / 給伺服器。
4. **伺服器回應**：回傳 HTML/CSS/JS。
5. **瀏覽器渲染**：解析並顯示，過程中再對 IP 用 ARP 找 MAC、經路由器轉送(IP 層繞送)等。
