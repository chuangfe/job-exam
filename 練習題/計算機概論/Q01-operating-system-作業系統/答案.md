# Q01 作業系統（OS）— 答案與解析

> 題目請見同資料夾的 `題目.md`。

---

**Q1（選擇）** 下列關於程序與執行緒的敘述，何者**錯誤**？

**答案：C**。執行緒共享所屬程序的 heap 與全域資料，各自獨立的只有「堆疊(stack)、暫存器、程式計數器」。A、B、D 皆正確。行程有獨立記憶體，切換要換頁表等，成本較高。

---

**Q2（選擇）** 程序從「執行(Running)」狀態因為發出 I/O 請求，會轉換到哪個狀態？

**答案：B**。發出 I/O 後 CPU 沒事做，程序去等 I/O 完成 → 進入 Waiting。I/O 完成後回到 Ready（不是直接 Running，要重新排隊）。時間片用完才是 Running→Ready。

---

**Q3（手算／簡答）** FCFS 與 SJF 平均等待時間。

等待時間 = 開始執行時間 − 到達時間（此處到達皆 0，故等待 = 前面所有人的 burst 加總）。

**FCFS（P1,P2,P3,P4）**
- P1 等 0；P2 等 8；P3 等 8+4=12；P4 等 8+4+9=21
- 平均 = (0+8+12+21)/4 = **41/4 = 10.25**

**SJF（依 burst 由小到大：P2(4)→P4(5)→P1(8)→P3(9)）**
- P2 等 0；P4 等 4；P1 等 4+5=9；P3 等 4+5+8=17
- 平均 = (0+4+9+17)/4 = **30/4 = 7.5**

重點：SJF 能得到最小平均等待時間，但需事先知道 burst（現實做不到），且長工作可能「飢餓(starvation)」。

**Q4（手算／簡答）** Round Robin（quantum=3）甘特圖與平均周轉時間。

執行順序（每次最多跑 3，沒跑完回隊尾）：

```
| P1 | P2 | P3 | P1 | P3 |
0    3    6    9   12   13
```
- P1：0-3 跑 3（剩 3），9-12 跑完 → 完成時間 12
- P2：3-6 跑完 → 完成時間 6
- P3：6-9 跑 3（剩 1），12-13 跑完 → 完成時間 13

周轉時間 = 完成時間 − 到達時間：P1=12、P2=6、P3=13
平均周轉 = (12+6+13)/3 = **31/3 ≈ 10.33**

RR 公平、回應快，適合分時系統；quantum 太小→切換成本高，太大→退化成 FCFS。

**Q5（選擇）** 關於虛擬記憶體與分頁，何者**正確**？

**答案：B**。
- (A) 錯：分頁是固定大小，消除外部碎裂，只會有少量「內部碎裂(internal fragmentation)」。
- (C) 錯：缺頁是「要的頁不在 RAM」，OS 從硬碟把它載進來即可，是正常機制（除非是非法存取）。
- (D) 錯：虛擬記憶體靠硬碟(swap)當後盾，可比實體 RAM 大，這正是它的賣點。

**Q6（手算／簡答）** 頁框數=3，參考字串 `7 0 1 2 0 3 0 4`，FIFO 與 LRU 缺頁次數。

**FIFO（先進先換掉最早進來的）**
```
7  -> [7]            缺
0  -> [7,0]          缺
1  -> [7,0,1]        缺
2  -> [0,1,2] 換7    缺
0  -> 命中
3  -> [1,2,3] 換0    缺
0  -> [2,3,0] 換1    缺
4  -> [3,0,4] 換2    缺
```
缺頁 = **7 次**

**LRU（換掉最久沒用到的）**
```
7  -> [7]            缺
0  -> [7,0]          缺
1  -> [7,0,1]        缺
2  -> [0,1,2] 換7(最久沒用)  缺
0  -> 命中，0變最近
3  -> [2,0,3] 換1    缺
0  -> 命中
4  -> [3,0,4] 換2    缺
```
缺頁 = **6 次**

LRU 通常優於 FIFO。注意 FIFO 有「Belady 異常」（頁框變多反而缺頁變多），LRU 不會。

**Q7（選擇／簡答）** 死結四個必要條件。

**四條件（必須同時成立）：**
1. **互斥(Mutual Exclusion)**：資源一次只能一人用。
2. **持有並等待(Hold and Wait)**：拿著資源還想要別的。
3. **不可剝奪(No Preemption)**：不能硬搶別人手上的資源。
4. **循環等待(Circular Wait)**：A 等 B、B 等 C、C 等 A，繞成一圈。

破解任一即可，例如「一次申請所有資源」破壞持有並等待；「資源編號、只能由小到大申請」破壞循環等待。

**Q8（選擇）** 磁頭像電梯來回掃的磁碟排程演算法。

**答案：C**。SCAN 像電梯來回掃，避免兩端飢餓。SSTF 永遠找最近的，可能讓遠端請求餓死；C-SCAN 是掃到底直接跳回起點（單向服務，等待較公平）。RR 是 CPU 排程，不是磁碟排程，屬干擾選項。
