# Q01 作業系統（OS）— 題目

## 觀念速補

- **作業系統是什麼**：管理硬體資源（CPU、記憶體、硬碟、I/O）並分配給程式用的「大管家」。你寫的 Node 程式不會自己去搶 CPU，而是 OS 排隊分時間給它。
- **程序（Process）vs 執行緒（Thread）**：
  - 程序 = 一個「正在執行的程式」，有獨立記憶體空間。類比：開一個獨立的 Node 進程（`node app.js`），互相看不到對方的變數。
  - 執行緒 = 程序裡的執行單位，**共享同一份記憶體**。類比：JS 的事件迴圈是單執行緒；Web Worker / Node worker_threads 才是多執行緒。
  - 共享記憶體很快但要小心「競爭條件（race condition）」，這就是為什麼要鎖（lock / mutex）。
- **行程狀態（Process State）**：新建(New) → 就緒(Ready，排隊等 CPU) → 執行(Running，正在用 CPU) → 等待/阻塞(Waiting/Blocked，例如等 I/O，像你 `await fetch()`) → 終止(Terminated)。
- **排程（Scheduling）**：CPU 只有幾顆，程式一堆，OS 要決定「下一個給誰跑」。這就是排程演算法。
- **記憶體管理**：實體 RAM 有限，但每支程式都以為自己獨佔一大塊記憶體 → 靠「虛擬記憶體 + 分頁」做出這個假象。類比：React 元件以為自己擁有整個畫面，其實是 React 幫你協調共用 DOM。
- **死結（Deadlock）**：兩個程序互相等對方放手，誰都不動。類比：兩個 await 互相等對方的 Promise resolve。

## 自我測驗

**Q1（選擇）** 下列關於程序與執行緒的敘述，何者**錯誤**？
- (A) 同一程序內的多個執行緒共享記憶體空間
- (B) 行程間切換（context switch）的成本通常比執行緒間切換高
- (C) 每個執行緒都有自己獨立、與其他執行緒完全隔離的堆積（heap）
- (D) 多執行緒程式需要同步機制來避免競爭條件

**Q2（選擇）** 程序從「執行(Running)」狀態因為發出 I/O 請求，會轉換到哪個狀態？
- (A) 就緒(Ready)　(B) 等待/阻塞(Waiting)　(C) 終止(Terminated)　(D) 新建(New)

**Q3（手算／簡答）** 有四個工作，到達時間皆為 0，CPU 執行時間（burst time）如下：

| 工作 | Burst |
|------|-------|
| P1 | 8 |
| P2 | 4 |
| P3 | 9 |
| P4 | 5 |

請分別用 **FCFS（先到先服務，順序 P1→P2→P3→P4）** 與 **SJF（最短工作優先）** 計算「平均等待時間」。

**Q4（手算／簡答）** 用 **Round Robin（RR）**，時間量子(time quantum)=3。三個工作到達時間皆 0，burst：P1=6、P2=3、P3=4。請畫出甘特圖並算平均周轉時間(turnaround time)。

**Q5（選擇）** 關於虛擬記憶體與分頁（paging），何者**正確**？
- (A) 分頁會造成「外部碎裂(external fragmentation)」
- (B) 分頁表(page table)負責把虛擬位址對應到實體位址
- (C) 缺頁(page fault)代表程式發生錯誤即將崩潰
- (D) 虛擬記憶體大小不可能大於實體記憶體

**Q6（手算／簡答）** 頁框數=3，存取頁面參考字串為 `7 0 1 2 0 3 0 4`。分別用 **FIFO** 與 **LRU** 計算缺頁次數。

**Q7（選擇／簡答）** 死結發生的四個必要條件是哪四個？只要打破其中一個就能避免死結。

**Q8（選擇）** 磁碟排程演算法中，磁頭從一端掃到另一端再折返（像電梯）的是？
- (A) FCFS　(B) SSTF（最短尋道優先）　(C) SCAN（電梯演算法）　(D) Round Robin
