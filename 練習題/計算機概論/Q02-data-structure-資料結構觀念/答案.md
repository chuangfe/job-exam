# Q02 資料結構觀念 — 答案與解析

> 題目請見同資料夾的 `題目.md`。

---

**Q1（選擇）** 以下何者時間複雜度為 O(log n)？

**答案：B**。二分搜尋每次砍一半 → O(log n)。A 是 O(n)、C 是 O(n²)、D 是 O(n²)。

---

**Q2（選擇）** min-heap 取出最小值的時間複雜度。

**答案：B**。「查看」最小值是 O(1)（就是根），但「取出」後要把最後一個元素移到根再下沉(heapify down)，高度為 log n → O(log n)。

---

**Q3（簡答）** 各結構「搜尋」平均時間複雜度。

| 結構 | 搜尋（平均） | 備註 |
|------|------|------|
| 未排序陣列 | O(n) | 要逐一比對 |
| 鏈結串列 | O(n) | 無法隨機存取 |
| 雜湊表 | O(1) | 最壞碰撞嚴重時 O(n) |
| 平衡 BST（如 AVL、紅黑樹） | O(log n) | 普通 BST 最壞退化 O(n) |

---

**Q4（選擇）** 未加權圖最短路徑（邊數最少）。

**答案：B**。BFS 一層層擴散，第一次抵達某節點時走過的邊數必為最少。DFS 不保證最短。（加權最短路徑要用 Dijkstra。）

---

**Q5（選擇）** 在同一個 bucket 用鏈結串列串起多個元素。

**答案：B**。鏈結法每格掛一條串列。開放定址（含線性探測、二次探測）則是碰撞時往其他空格放。負載因子(load factor)過高時效能會降。

---

**Q6（手算／簡答）** 由中序與前序還原二元樹並寫出後序。

步驟：前序第一個是根 → **A**。中序中 A 左邊 `D B E` 是左子樹，右邊 `F C` 是右子樹。
- 左子樹前序 `B D E`：根 B；中序 `D B E` → D 左、E 右。
- 右子樹前序 `C F`：根 C；中序 `F C` → F 在左、右為空。

樹形：
```
        A
       / \
      B   C
     / \  /
    D   E F
```
**後序(左→右→根)** = `D E B F C A`

---

**Q7（手算／簡答）** 中序↔後序轉換。

- `A + B * C` → 乘法優先 → `A + (B*C)` → 後序 **`A B C * +`**
- `A B + C *` → 讀到 `+`：`(A+B)`；讀到 `*`：`(A+B)*C` → 中序 **`(A + B) * C`**

技巧：後序就是「運算元先放、運算子放在兩個運算元之後」。用堆疊計算後序最直觀。

---

**Q8（選擇）** 平均時間複雜度為 O(n log n) 的排序。

**答案：C**。Quick Sort 平均 O(n log n)（最壞 O(n²)）。Merge Sort、Heap Sort 也是 O(n log n)。A、B、D 平均皆 O(n²)。
