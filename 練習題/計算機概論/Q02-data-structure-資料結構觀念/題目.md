## 觀念速補

- **Big-O 是什麼**：描述「資料量變大時，時間/空間成長多快」，忽略常數。你在 React 裡寫 `array.map` 是 O(n)；雙層巢狀迴圈是 O(n²)。考試常考「最壞情況」。
- **常見複雜度排序**（由快到慢）：O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)。
- **陣列 vs 鏈結串列**：陣列隨機存取 O(1)、中間插入 O(n)；鏈結串列插入 O(1)、查找 O(n)。類比：陣列像有編號的格子，鏈結串列像每個人手牽手。
- **堆疊(Stack) vs 佇列(Queue)**：Stack 後進先出(LIFO)，像瀏覽器上一頁、函式呼叫堆疊；Queue 先進先出(FIFO)，像 Node 的 task queue。
- **樹/BST/Heap**：
  - BST(二元搜尋樹)：左<根<右，搜尋平均 O(log n)，最壞（退化成鏈）O(n)。
  - Heap(堆積)：完全二元樹，最大堆根最大，用來做優先佇列；不是排序好的。
- **圖的走訪**：DFS 用堆疊/遞迴往深處鑽；BFS 用佇列一層一層擴散（最短路徑/層級用 BFS）。類比：DFS 像 DOM 遞迴往下找子節點，BFS 像一層層掃同輩。
- **雜湊(Hash)**：用 key 算出 index 直接存取，平均 O(1)。類比：JS 的物件 / Map。**碰撞**就是兩個 key 算到同一格，要用鏈結法(chaining)或開放定址(open addressing)解決。

## 自我測驗

**Q1（選擇）** 以下何者時間複雜度為 O(log n)？
- (A) 線性搜尋一個未排序陣列
- (B) 在已排序陣列做二分搜尋(binary search)
- (C) 氣泡排序(bubble sort)
- (D) 走訪一個 n×n 矩陣

**Q2（選擇）** 對含 n 個元素的最小堆積(min-heap)，取出最小值並維持堆積性質的時間複雜度為？
- (A) O(1)　(B) O(log n)　(C) O(n)　(D) O(n log n)

**Q3（簡答）** 平均情況下，比較「陣列」「鏈結串列」「雜湊表」「平衡 BST」的「搜尋」時間複雜度。

**Q4（選擇）** 圖的走訪中，欲求「未加權圖的最短路徑（邊數最少）」，最適合用？
- (A) DFS　(B) BFS　(C) 二分搜尋　(D) 拓樸排序

**Q5（選擇）** 雜湊碰撞(collision)的解決方法中，「在同一個 bucket 用鏈結串列把多個元素串起來」稱為？
- (A) 開放定址(open addressing)　(B) 鏈結法(separate chaining)　(C) 線性探測(linear probing)　(D) 再雜湊(rehashing)

**Q6（手算／簡答）** 已知某二元樹的**中序(inorder)** = `D B E A F C`、**前序(preorder)** = `A B D E C F`，請還原這棵樹並寫出**後序(postorder)**。

**Q7（手算／簡答）** 將中序運算式 `A + B * C` 轉成後序(postfix)；再把後序 `A B + C *` 轉回中序。

**Q8（選擇）** 下列排序演算法，平均時間複雜度為 O(n log n) 的是？
- (A) 氣泡排序(Bubble Sort)　(B) 插入排序(Insertion Sort)　(C) 快速排序(Quick Sort)　(D) 選擇排序(Selection Sort)
