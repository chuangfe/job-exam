# Q25 閉包與作用域（var vs let）—— 答案

## 輸出

```
var: 3
var: 3
var: 3
let: 0
let: 1
let: 2
```

## 解析

- `var` 沒有區塊作用域，三個 callback 共享同一個 `i`；迴圈跑完時 `i` 已經是 3，等到 `setTimeout` 執行時讀到的都是 3。
- `let` 每次迭代都會產生一個全新的綁定 `j`，因此各個 callback 捕捉到的是當下那一輪的值（0、1、2）。
- `setTimeout` 屬於非同步任務，會等同步程式全部跑完才執行，所以六行輸出全部排在最後（兩個迴圈的同步部分先跑完）。

這是經典必考題：考的是 `var` 函式作用域 vs `let` 區塊作用域，以及閉包捕捉變數的時機。
