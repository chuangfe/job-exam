> - 可執行的程式題 → `題目.js`（題目敘述 + stub + 測試，未實作時 `node 題目.js` 會拋錯）與 `答案.js`（完整解答 + 相同測試，`node 答案.js` 通過後印 `✅ 通過`）。

## 考試手寫技巧

1. **先寫終止條件**：遞迴題第一行就寫 base case。
2. **邊界條件先想**：空陣列、長度 0/1、null、負數、相等的情況。
3. **草稿模擬小案例**：用 n=2 或 3 個元素手動跑一遍。
4. **變數命名清楚**：`left/right`、`prev/curr/next`、`i/j`、`minIdx`。
5. **善用雙指標與交換**：`[a, b] = [b, a]` 是 JS 交換利器。
6. **講複雜度加分**：寫完補一句時間／空間複雜度。
7. **資料結構先畫圖**：鏈結串列、BST 先畫節點與箭頭。
8. **不確定內建語法就手寫**：邏輯對最重要。
9. **程式閱讀題逐行標值**：注意 var/let、傳值/傳參考、非同步順序。
10. **時間分配**：選擇題快狠準，時間留給手寫與閱讀題。
