# Q29 提升（Hoisting）與作用域 —— 答案

## 輸出

```
undefined
foo 被呼叫
ReferenceError
```

## 解析

- `var x` 的**宣告**被提升到作用域頂端，但**賦值**留在原地，所以先印出 `undefined`（已宣告、未賦值）。
- **函式宣告**整個被提升，所以可以在定義之前就呼叫 `foo()`，印出「foo 被呼叫」。
- `let y` 也會提升，但處於**暫時死區（TDZ, Temporal Dead Zone）**，在賦值之前存取會丟出 `ReferenceError`（而不是 `undefined`）——這正是 `let`/`const` 與 `var` 的關鍵差異。
