# Q05 Express REST API：完整可跑的 Tasks CRUD

REST 的核心：**用 HTTP method 表達動作、用 URL 表達資源**。

| 動作 | Method | URL | 對應前端 |
|---|---|---|---|
| 查全部 | GET | `/tasks` | `fetch('/tasks')` |
| 查單筆 | GET | `/tasks/:id` | `fetch('/tasks/1')` |
| 新增 | POST | `/tasks` | `fetch('/tasks', {method:'POST', body})` |
| 更新 | PUT | `/tasks/:id` | `fetch('/tasks/1', {method:'PUT', body})` |
| 刪除 | DELETE | `/tasks/:id` | `fetch('/tasks/1', {method:'DELETE'})` |

## 目標

建立 tasks 資源的完整 CRUD，含路由、中介層、請求驗證、**統一錯誤處理 middleware**。先用記憶體陣列當資料（Q06 換成真資料庫）。

## 要安裝的套件

```bash
mkdir node-practice && cd node-practice
npm init -y
npm pkg set type=module        # 讓你能用 import/export（和前端一致）
npm install express
```

> 全文使用 ES Module（`import`），和你在 React 寫的語法一致。若你的環境只能用 CommonJS，把 `import x from 'y'` 換成 `const x = require('y')`、`export` 換成 `module.exports` 即可。

## 步驟

1. 建立 `app.js`，匯出一個 `createApp()` 函式（回傳組裝好的 express app，方便 Q08 測試時 import）。
2. 掛上 `express.json()` 中介層解析 JSON body。
3. 寫一個簡易請求紀錄 middleware（log `req.method` 與 `req.url`）。
4. 寫 `validateTask` 驗證 middleware：檢查 `title` 是非空字串，否則 `next(new AppError(..., 400))`。
5. 寫 `wrap` 樣板包住 async handler，自動把錯誤轉給錯誤 middleware。
6. 實作 5 條路由：GET 全部、GET 單筆、POST 新增、PUT 更新、DELETE 刪除。
7. 加 404 fallback 與**統一錯誤處理 middleware（4 個參數，放最後）**。
8. 只有在非測試環境才 `app.listen(3000)`。

## 提示

- `express.json()` 中介層：解析 JSON body（沒有它 `req.body` 會是 undefined）。
- middleware 簽名 `(req, res, next)`：呼叫 `next()` 往下傳，呼叫 `next(err)` 跳到錯誤處理。
- 錯誤處理 middleware 簽名有 **4 個參數** `(err, req, res, next)`，Express 靠參數數量辨識它。
- 自訂錯誤帶 `statusCode`（沿用 Q04 的 `AppError`），讓錯誤 middleware 直接決定狀態碼。
- 狀態碼語意：201 新增成功、204 刪除成功無內容、400 請求格式錯、404 找不到、500 伺服器內部錯。

## 測試你的 API

```bash
node app.js
# 另開終端機：
curl http://localhost:3000/tasks
curl -X POST http://localhost:3000/tasks -H 'Content-Type: application/json' -d '{"title":"新任務"}'
curl -X PUT http://localhost:3000/tasks/1 -H 'Content-Type: application/json' -d '{"title":"改過","done":true}'
curl -X DELETE http://localhost:3000/tasks/2
```

## 重點

- **middleware 是有順序的**：`express.json()` 要在路由前、錯誤處理要在最後。
- **`wrap` 樣板很重要**：Express 4 不會自動接住 async handler 丟出的錯誤，少了它 app 會卡住。
- **驗證放 middleware**：把「檢查」和「業務邏輯」分離，乾淨且可重用。
