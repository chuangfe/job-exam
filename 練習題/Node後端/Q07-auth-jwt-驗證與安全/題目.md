# Q07 驗證與安全：bcrypt + JWT + 環境變數

API 開出去後，要回答兩個問題：**你是誰（authentication）** 和 **你能做什麼（authorization）**。

## 觀念：JWT 登入流程（前端視角）

```
1. 使用者 POST /login 帶帳密
2. 後端驗證帳密 → 簽發一個 JWT（一串編碼字串）
3. 前端把 JWT 存起來（localStorage / cookie）
4. 之後每次 fetch 都帶 header: Authorization: Bearer <token>
5. 後端用密鑰驗證 token 沒被竄改、沒過期 → 放行
```

> JWT = 一張「蓋了伺服器防偽章的識別證」。後端不需查 DB 就能驗證它是真的（因為有簽章），這就是它的價值。

## 要安裝的套件

```bash
npm install bcryptjs jsonwebtoken dotenv
```

---

## 子題 7-1：密碼雜湊（bcrypt）

**目標**：使用者註冊時，**永遠不要存明碼密碼**，要存雜湊後的結果。

**步驟**：寫 `hashPassword(plain)`（用 `bcrypt.hash`，saltRounds = 10）與 `verifyPassword(plain, hashed)`（用 `bcrypt.compare`）。

**提示**：`hash` 加鹽雜湊、`compare` 比對（登入時用）。

---

## 子題 7-2：JWT 簽發與驗證 middleware

**目標**：實作 `/login` 發 token、以及保護路由的 `authMiddleware`。

**步驟**：
1. `POST /login`：找使用者 → `bcrypt.compare` 驗密碼 → 失敗回 401（帳號錯與密碼錯都回一樣訊息）。
2. 成功則 `jwt.sign({ sub, email }, JWT_SECRET, { expiresIn: '1h' })`，回 `{ token }`。
3. `authMiddleware`：從 `Authorization: Bearer <token>` 取 token → `jwt.verify` → 成功把 payload 掛到 `req.user`，失敗回 401。
4. 套用到受保護路由（例如 `GET /me`）。

**提示**：`jwt.sign(payload, secret, opts)` 發、`jwt.verify(token, secret)` 驗。

---

## 子題 7-3：環境變數

**目標**：把密鑰、DB 連線字串等敏感資訊抽離程式碼，**不進 git**。

**步驟**：
1. `npm install dotenv`，建立 `.env`（放 `JWT_SECRET`、`DATABASE_URL`、`PORT`）。
2. 在程式最上方 `import 'dotenv/config'`（要最早執行）。
3. 啟動時檢查必要環境變數存在，缺了就 throw。
4. 把 `.env`、`node_modules/`、`*.db` 加進 `.gitignore`，並附一份 `.env.example`。

**提示**：`npm install dotenv`，建立 `.env`，並加進 `.gitignore`。

---

## 重點

- 401 = 未認證（沒登入 / token 壞）；403 = 已認證但沒權限。
- token 要設 `expiresIn`，過期就得重新登入（或用 refresh token，進階）。
- bcrypt 是**單向**的（無法反解），且自動加鹽 → 即使資料庫外洩，攻擊者也拿不到原始密碼。**絕不可用 MD5/SHA1 存密碼**。
- 程式碼進 git、**機密不進 git**。正式環境（Vercel、Render、AWS）由平台的環境變數面板注入。
- `authMiddleware` 把 `req.user` 掛上去，後續 handler 就知道「現在是誰」。
