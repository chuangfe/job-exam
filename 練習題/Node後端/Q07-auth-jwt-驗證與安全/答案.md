# Q07 驗證與安全：bcrypt + JWT + 環境變數（參考解答）

## 7-1：密碼雜湊（bcrypt）

```js
import bcrypt from 'bcryptjs';

// 註冊：把明碼變雜湊存進 DB
async function hashPassword(plain) {
  const saltRounds = 10; // 成本因子，越大越安全但越慢
  return bcrypt.hash(plain, saltRounds);
}

// 登入：比對使用者輸入 vs DB 裡的雜湊
async function verifyPassword(plain, hashed) {
  return bcrypt.compare(plain, hashed); // true / false
}

const hashed = await hashPassword('mypassword');
console.log(await verifyPassword('mypassword', hashed)); // true
console.log(await verifyPassword('wrong', hashed));      // false
```

> bcrypt 是**單向**的（無法反解），且自動加鹽（同一密碼每次雜湊結果都不同）→ 即使資料庫外洩，攻擊者也拿不到原始密碼。**絕不可用 MD5/SHA1 存密碼**。

## 7-2：JWT 簽發與驗證 middleware

```js
import jwt from 'jsonwebtoken';
import bcrypt from 'bcryptjs';

const JWT_SECRET = process.env.JWT_SECRET; // 見 7-3 環境變數

// 假使用者表（正式環境來自 DB）
const users = [
  { id: 1, email: 'ian@example.com', passwordHash: bcrypt.hashSync('1234', 10) },
];

// POST /login
app.post('/login', wrap(async (req, res) => {
  const { email, password } = req.body;
  const user = users.find((u) => u.email === email);
  // 注意：帳號錯與密碼錯都回一樣的訊息，避免洩漏「帳號是否存在」
  if (!user || !(await bcrypt.compare(password, user.passwordHash))) {
    throw new AppError('帳號或密碼錯誤', 401);
  }
  const token = jwt.sign({ sub: user.id, email: user.email }, JWT_SECRET, {
    expiresIn: '1h',
  });
  res.json({ token });
}));

// 保護路由用的 middleware
function authMiddleware(req, res, next) {
  const header = req.headers.authorization || '';
  const token = header.startsWith('Bearer ') ? header.slice(7) : null;
  if (!token) return next(new AppError('未提供認證 token', 401));
  try {
    req.user = jwt.verify(token, JWT_SECRET); // 驗章 + 驗過期；成功後把 payload 掛到 req
    next();
  } catch {
    next(new AppError('token 無效或已過期', 401));
  }
}

// 套用：只有帶有效 token 才能存取
app.get('/me', authMiddleware, (req, res) => {
  res.json({ id: req.user.sub, email: req.user.email });
});
```

### 前端搭配方式

```js
const { token } = await fetch('/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email, password }),
}).then((r) => r.json());

// 之後的請求帶上 token
await fetch('/me', { headers: { Authorization: `Bearer ${token}` } });
```

## 7-3：環境變數

### `.env`（絕對不要 commit）

```bash
JWT_SECRET=請用一段夠長的隨機字串_例如用_openssl_rand_hex_32_產生
DATABASE_URL=postgres://user:pass@localhost:5432/mydb
PORT=3000
```

### 在程式最上方載入（要最早執行）

```js
import 'dotenv/config';

const port = process.env.PORT || 3000;
if (!process.env.JWT_SECRET) {
  throw new Error('缺少必要環境變數 JWT_SECRET'); // 啟動時就擋下，別等出事
}
```

### `.gitignore`

```bash
node_modules/
.env
*.db
```

## 重點

- 401 = 未認證（沒登入 / token 壞）；403 = 已認證但沒權限。
- token 要設 `expiresIn`，過期就得重新登入（或用 refresh token，進階）。
- `authMiddleware` 把 `req.user` 掛上去，後續 handler 就知道「現在是誰」。
- 程式碼進 git、**機密不進 git**。正式環境由平台的環境變數面板注入。
- 慣例附一份 `.env.example`（只有 key、沒有值）讓協作者知道要設哪些變數。
