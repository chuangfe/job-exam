# Q06 資料庫：SQL 基礎 + 用 Node 串接 SQLite

記憶體陣列重啟就消失，真實後端要把資料存進資料庫。先補 SQL 基礎，再用 Node 串接。

## Part A. SQL 基礎練習

假設兩張表：

```sql
CREATE TABLE users (
  id    INTEGER PRIMARY KEY,
  name  TEXT NOT NULL,
  email TEXT UNIQUE NOT NULL
);

CREATE TABLE tasks (
  id      INTEGER PRIMARY KEY,
  user_id INTEGER NOT NULL,
  title   TEXT NOT NULL,
  done    INTEGER DEFAULT 0,        -- SQLite 用 0/1 當布林
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

**目標**：練 CRUD + JOIN（後端最常寫的查詢）。自己先動手寫出：

1. 新增一個 user 與一筆 task。
2. 查所有未完成 task、依 id 倒序取前 10 筆、依 user_id 聚合計數。
3. 把某筆 task 標記為完成、刪除某筆 task。
4. `INNER JOIN`：把任務和它的擁有者名字接起來。
5. `LEFT JOIN`：列出每個使用者及其任務數（沒有任務的人也要出現）。

## Part B. 用 Node 串接 SQLite（含參數化查詢）

**目標**：把 Q05 的記憶體陣列換成真資料庫，並用**參數化查詢**防止 SQL Injection。

### 要安裝的套件

```bash
npm install better-sqlite3   # 同步 API、零設定、最適合練習
```

### 步驟

1. 建立 `db.js`，用 `better-sqlite3` 開一個 `:memory:` DB（正式改成 `./data.db`）。
2. `db.exec` 建 `tasks` 表（`id` AUTOINCREMENT、`title`、`done`）。
3. 匯出 `taskRepo` 物件，含 `all / find / create / update / remove`，**全部用 `?` 參數化查詢**。
4. 把 Q05 的 handler 改用 `taskRepo`（路由層只管 HTTP）。

### 提示

- **絕對不要**用字串拼 SQL（`` `WHERE id = ${id}` ``）→ 這是 SQL Injection 漏洞。
- 改用 `?` 佔位符，把值當參數傳進去，資料庫會自動做跳脫。
- `INNER JOIN` 只留兩邊都有對應的列；`LEFT JOIN` 保留左表全部、右表沒對應補 NULL。
- 想練更接近職場的 DB，可改用 PostgreSQL（`pg`）：佔位符是 `$1, $2`，且 API 是非同步。

## 重點

- **參數化查詢**是後端安全的底線，面試常考——務必能說出「為什麼能防 injection」（值與 SQL 語法分離，值永遠被當資料而非指令）。
- 把資料存取邏輯收進 `repo`（Repository 模式），路由層只管 HTTP，**關注點分離**讓測試更好寫。
