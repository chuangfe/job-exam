# Q06 資料庫：SQL 基礎 + 用 Node 串接 SQLite（參考解答）

## Part A. SQL 基礎

```sql
-- C 新增
INSERT INTO users (name, email) VALUES ('Ian', 'ian@example.com');
INSERT INTO tasks (user_id, title) VALUES (1, '寫 SQL 練習');

-- R 查詢
SELECT * FROM tasks WHERE done = 0;                 -- 條件查詢
SELECT * FROM tasks ORDER BY id DESC LIMIT 10;      -- 排序 + 分頁
SELECT user_id, COUNT(*) AS cnt FROM tasks GROUP BY user_id; -- 聚合

-- U 更新
UPDATE tasks SET done = 1 WHERE id = 1;

-- D 刪除
DELETE FROM tasks WHERE id = 1;

-- JOIN：把任務和它的擁有者名字接起來（後端常見需求）
SELECT t.id, t.title, t.done, u.name AS owner
FROM tasks AS t
INNER JOIN users AS u ON t.user_id = u.id
WHERE u.email = 'ian@example.com';

-- LEFT JOIN：列出每個使用者及其任務數（沒有任務的人也要出現）
SELECT u.name, COUNT(t.id) AS task_count
FROM users AS u
LEFT JOIN tasks AS t ON t.user_id = u.id
GROUP BY u.id;
```

## Part B. `db.js`（用 Node 串接 SQLite）

```js
import Database from 'better-sqlite3';

const db = new Database(':memory:'); // 練習用記憶體 DB；正式改成 './data.db'

db.exec(`
  CREATE TABLE IF NOT EXISTS tasks (
    id    INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    done  INTEGER DEFAULT 0
  );
`);

export const taskRepo = {
  all() {
    return db.prepare('SELECT * FROM tasks ORDER BY id').all();
  },
  find(id) {
    // ✅ 參數化：? 由 driver 安全代入，杜絕 injection
    return db.prepare('SELECT * FROM tasks WHERE id = ?').get(id);
  },
  create(title) {
    const info = db.prepare('INSERT INTO tasks (title) VALUES (?)').run(title);
    return this.find(info.lastInsertRowid);
  },
  update(id, { title, done }) {
    db.prepare('UPDATE tasks SET title = ?, done = ? WHERE id = ?')
      .run(title, done ? 1 : 0, id);
    return this.find(id);
  },
  remove(id) {
    return db.prepare('DELETE FROM tasks WHERE id = ?').run(id).changes; // 受影響列數
  },
};
```

## 對比：危險 vs 安全

```js
// ❌ 危險：若 id = "1 OR 1=1"，會撈出全部資料
db.prepare(`SELECT * FROM tasks WHERE id = ${id}`).get();

// ✅ 安全：參數化查詢
db.prepare('SELECT * FROM tasks WHERE id = ?').get(id);
```

## PostgreSQL 版本（更接近職場，用 `pg`）

佔位符是 `$1, $2`，且 API 是非同步：

```js
import pg from 'pg';
const pool = new pg.Pool({ connectionString: process.env.DATABASE_URL });
const { rows } = await pool.query('SELECT * FROM tasks WHERE id = $1', [id]);
```

## 重點

- **參數化查詢**是後端安全的底線（值與 SQL 語法分離，值永遠被當資料而非指令）。
- 把資料存取邏輯收進 `repo`（Repository 模式），路由層只管 HTTP，**關注點分離**讓測試更好寫。
- 前端類比：JOIN 就像你在前端把 `users` 和 `tasks` 兩個陣列用 `userId` `find`/`map` 接起來，但資料庫做得更快更省記憶體。
