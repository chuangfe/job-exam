# Q09 Capstone 專案：個人筆記 API

把前面所有東西串起來，做一個可以**放進履歷 / 作品集**的完整後端服務。

## 專案規格

> 一個多使用者的「個人筆記」服務。每個使用者註冊登入後，只能對**自己的**筆記做 CRUD。

**技術棧**：Express + SQLite（或 PostgreSQL）+ JWT + bcrypt + Jest/Supertest + dotenv。

## 功能清單

- [ ] 使用者註冊（密碼用 bcrypt 雜湊存）
- [ ] 使用者登入（驗證後發 JWT）
- [ ] 筆記 CRUD（建立 / 列表 / 查單筆 / 更新 / 刪除）
- [ ] **權限隔離**：A 使用者不能讀寫 B 使用者的筆記
- [ ] 請求驗證（必填欄位、型別）
- [ ] 統一錯誤處理 middleware
- [ ] 環境變數管理機密
- [ ] 單元測試 + 整合測試（涵蓋成功與失敗路徑）

## API 設計

| Method | URL | 需要登入 | 說明 |
|---|---|---|---|
| POST | `/auth/register` | ❌ | 註冊，body: `{ email, password }` |
| POST | `/auth/login` | ❌ | 登入，回 `{ token }` |
| GET | `/notes` | ✅ | 列出**自己的**筆記（支援 `?q=關鍵字` 搜尋） |
| POST | `/notes` | ✅ | 新增筆記，body: `{ title, content }` |
| GET | `/notes/:id` | ✅ | 查單筆（非本人 → 404） |
| PUT | `/notes/:id` | ✅ | 更新 |
| DELETE | `/notes/:id` | ✅ | 刪除 |

## 資料表設計

```sql
CREATE TABLE users (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  email         TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  created_at    TEXT DEFAULT (datetime('now'))
);

CREATE TABLE notes (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id    INTEGER NOT NULL,
  title      TEXT NOT NULL,
  content    TEXT DEFAULT '',
  created_at TEXT DEFAULT (datetime('now')),
  updated_at TEXT DEFAULT (datetime('now')),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 實作步驟（建議順序）

1. **建專案骨架**：`npm init`、裝套件、設 `.gitignore` 與 `.env`。
2. **資料層**：寫 `db.js` 建表、`userRepo` / `noteRepo`（全部用參數化查詢）。
3. **認證**：`/auth/register`（bcrypt 雜湊）→ `/auth/login`（驗證 + 簽 JWT）。
4. **authMiddleware**：驗 token、把 `req.user` 掛上。
5. **筆記 CRUD**：每個查詢都帶 `WHERE user_id = req.user.sub`，達成權限隔離。
6. **錯誤處理**：自訂 `AppError`、統一 error middleware、`wrap` 包 async。
7. **測試**：先寫整合測試跑通主流程，再補失敗路徑（401/403/404/400）。
8. **README**：寫安裝步驟、API 文件、`curl` 範例（作品集加分）。

## 驗收標準（Definition of Done）

- [ ] 未帶 token 存取 `/notes` → 回 **401**
- [ ] A 使用者用自己的 token 存取 B 的筆記 id → 回 **404**（不可回 200 或 403 洩漏存在）
- [ ] 註冊後資料庫裡存的是**雜湊**，不是明碼
- [ ] 登入錯誤的密碼 → 回 **401**，且訊息不區分「帳號錯」與「密碼錯」
- [ ] 缺必填欄位 → 回 **400** 並有清楚錯誤訊息
- [ ] `npm test` 全綠，且涵蓋認證、CRUD、權限隔離、錯誤路徑
- [ ] 機密全在 `.env`，repo 內 grep 不到任何明碼密鑰
- [ ] README 能讓陌生人 5 分鐘內跑起來
